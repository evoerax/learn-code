# 软件工程

研究如何构建与维护**大型软件系统**：不只是“写代码”，而是**可长期演化、可观测、可回滚、可协作**。

---

## 子方向

| 子题 | 典型问题 |
| :--- | :--- |
| **架构** | 模块化、边界、依赖治理、演进式架构 |
| **质量与验证** | 测试、静态/动态分析、形式化方法（与理论交叉） |
| **交付与运维** | CI/CD、发布策略、金丝雀、可观测性（日志/指标/追踪） |
| **平台工程** | 内部开发者平台、golden paths、自助服务与治理 |
| **过程与协作** | 需求、代码评审、技术债度量 |

---

## 2026 常见叙事

工业界常把这条线表述为 **platform engineering**、**developer experience** 与 **可持续交付**；与 Thoughtworks 等机构的年度趋势（[Looking Glass](https://www.thoughtworks.com/insights/looking-glass)）同屏出现。

---

## 与 AI 的交叉

大模型改变的是**编写与审查代码的方式**，但**流水线、策略、治理、供应链安全**仍是软件工程问题。
