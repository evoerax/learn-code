# 1980s — 面向对象革命

1980 年代将面向对象编程从研究实验室带入了生产代码库。C++ 为世界上最流行的系统语言添加了对象。Objective-C 对 Smalltalk 风格的消息传递做了同样的事。与此同时，函数式编程走向成熟，逻辑编程进入主流，个人计算机革命创造了全新的软件类型——以及程序员。

## 背景

| | |
|---|---|
| **时代特征** | OOP 普及、个人计算机、工作站 |
| **代表硬件** | IBM PC（1981）、Apple Macintosh（1984）、Sun 工作站 |
| **里程碑** | Smalltalk-80 公开发布；C++ 首次描述（1979–1983） |
| **核心挑战** | 管理百万行代码库；GUI 编程；实时系统 |

<Callout type="info">
  Bjarne Stroustrup 在 1979 年为其剑桥博士论文编写程序时创造了"带类的 C"。他需要 Simula 的抽象能力来完成一个模拟程序，但负担不起 Simula 的性能开销。结果就是 C++。
</Callout>

## 核心主题

**OOP 走向主流。** C++ 起初颇有争议——"带类的 C"对纯粹主义者来说像是妥协。但它解决了一个真实问题：大型团队需要围绕数据和行为来组织代码，而 C++ 让他们可以在不放弃 C 性能的情况下做到这一点。到 1990 年，OOP 已成为新系统软件的主流范式。

**可靠性作为设计目标。** Ada 是为美国国防部设计的，旨在解决军事系统中的软件故障问题。强静态类型、范围检查数组、异常处理和并发原语都是强制性的，而非可选的。Ada 影响了整个领域对"正确"软件意义的思考。

**Erlang 的悄悄革命。** 爱立信需要永不宕机的软件——电话交换机不能重启。Erlang 的答案是：通过消息传递通信的轻量级进程（actors）、重启失败进程的监督树，以及可以在不停止系统的情况下更新运行中系统的热代码加载。这种"让它崩溃"的哲学将在 2010 年代的云计算中重新出现。

**脚本语言填补了空白。** Perl 以"简单的事情应该简单，复杂的事情应该可能"为座右铭。对于需要处理文本文件和自动化 Unix 任务的系统管理员来说，Perl 是一个启示——强大的正则表达式、灵活的数据结构，以及对类型声明毫无坚持。它催生了现代脚本语言类别。

**个人计算机创造了新用户。** Apple Macintosh 的 GUI 需要事件驱动编程。HyperCard（1987）让非程序员也能创建交互式应用程序。"程序员"和"用户"之间的界限开始模糊。

## 展望未来

1980 年代结束时，OOP 已经牢固确立，而函数式编程仍然主要停留在学术界。1990 年代将带来互联网，迫使每种语言处理网络、并发和 Web 内容——并将解释型脚本语言提升到与编译系统语言同等的地位。
