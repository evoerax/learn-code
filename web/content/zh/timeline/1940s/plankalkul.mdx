# Plankalkül

> _第一门高级编程语言的纯理论设计。_

## 概览

## 诞生背景

Plankalkül 由德国工程师康拉德·楚泽于 1943 年至 1948 年间在柏林设计，创作时期横跨第二次世界大战及战后岁月。楚泽在研制 Z3 和 Z4 计算机——世界上最早的可编程计算机——的同时，开发了这门语言。

与同时期美国开发的 ENIAC Short Code 不同，Plankalkül 始终停留在纯理论层面。楚泽撰写了长达 100 页的语言规格说明书，但由于战后德国的技术限制和破坏，他设计程序时没有任何计算机能够执行这些代码。

"Plankalkül" 这个名称源自德语："Plan"（计划）和 "Kalkül"（演算），合起来意为"计划演算"或"形式化计划系统"。

## 设计理念

Plankalkül 引入的概念要比主流语言早数十年才出现：

- **结构化数据类型**：记录（称为"结构"）、数组和嵌套数据结构
- **复杂数据操作**：不仅能处理标量值，还能对整个数组进行操作
- **子程序和函数**：代码复用块的早期概念
- **形式语义学**：楚泽试图为他的语言建立严格的数学基础

<Callout type="default" emoji="💡">
  Plankalkül 包含图形输出组件，用于显示算法结果——楚泽预见了编程既是计算也是可视化的未来。
</Callout>

这门语言在当时非常先进，融合了数据抽象和算法验证的思想，而这些要到 1970 和 1980 年代才成为主流。

## 代码示例

```plankalkul
P1 × C[5:5] → C[5:5]
R1 := R2 × R1 + R3
R1.0 := R2.0 ⊗ R3.0
```

这个例子（来自楚泽原始记号法）展示了：
- `P1` 定义了一个程序，接受一个 5×5 的复数数组（`C[5:5]`）
- `R1 := R2 × R1 + R3` 展示了对寄存器的算术运算
- `R1.0 := R2.0 ⊗ R3.0` 展示了对数组元素的操作

## 影响与遗产

**灵感来源：** 数学符号法、希尔伯特的形式系统、楚泽 Z1/Z2/Z3 计算机的工作

**影响语言：** ALGOL、Pascal、Ada、现代结构化编程

Plankalkül 对后续语言的影响更多是启发性的而非直接的。当 1950 年代末 ALGOL 的设计者们得知楚泽的早期工作时，他们认识到两者设计目标的许多相似之处。这门语言证明了：

1. 高级编程概念可以被形式化、数学化
2. 数据结构可以独立于机器表示进行设计
3. 算法可以用独立于机器的记号法表达

Plankalkül 直到 1970 年代才被实现——当时苏黎世联邦理工学院的 Heinz Rutishauser 团队根据楚泽的规格说明创建了一个可工作的解释器。但到那时，它的许多创新已被其他语言设计者独立重新发现。
