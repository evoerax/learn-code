## 诞生背景

Short Code（也称为 Brief Code）由威廉·F·布鲁克和贝蒂·霍尔顿于 1949 年在雷明顿兰德公司为 UNIVAC I 计算机开发。它源于为 BINAC 计算机创建的早期 Short Code，但为 UNIVAC I 进行了重大改进。

ENIAC 需要人工"编译器"手动编译，而 Short Code 引入了一种更系统化的方法：程序用简化的符号记号法编写，由解释器处理。每条数学表达式都被转换为双地址码格式。

<Callout type="default" emoji="💡">
  Short Code 所需的机器指令大约是手写机器码的 50 倍，但程序员编写代码的速度大约快 50 倍——这是一种革命性的权衡。
</Callout>

## 设计理念

Short Code 代表了早期编程的一种务实方法：

- **符号表示**：用数学简写表示变量和操作
- **双地址格式**：每条指令指定操作和两个内存位置
- **解释执行**：驻留解释器将 Short Code 处理成机器指令

这种语言针对的是人类生产力而非机器效率——这一主题贯穿了编程语言的演进。

## 代码示例

```shortcode
X1 = 5000
X2 = X1 + 200
X3 = X2 / 200
```

这个例子展示了 Short Code 的方法：简单的数学语句，用比机器码更接近代数的记号法表示。程序员用这种符号形式编写代码；解释器负责翻译成 UNIVAC 的机器语言。

## 影响与遗产

**灵感来源：** ENIAC Short Code、数学符号法

**影响语言：** FLOW-MATIC、COBOL

Short Code 确立了一个重要原则：编程应该优先考虑人类的理解而非机器的效率。这一理念直接影响 Grace Hopper 开发 FLOW-MATIC 的工作，进而影响 COBOL——历史上最持久的语言之一。Short Code 表明，即使更慢，高级抽象也可以让计算机对更广泛的用户变得更容易使用。
