## 诞生背景

ENIAC Short Code 由约翰·莫克利和普雷斯珀·埃克特于 1946 年在宾夕法尼亚大学开发。它是第一门真正在电子计算机——ENIAC（电子数值积分计算机）上运行的编程语言。

在 ENIAC Short Code 出现之前，计算机必须通过物理开关和电缆直接以机器码编程。Short Code 引入了一种革命性的概念：用符号形式表示数学表达式，让计算机能够解释并执行。

这门语言实际上是一种解释系统——程序用数学符号编写，然后由一组女性程序员（被称为"ENIAC 程序员"）手动转换为 ENIAC 能执行的机器码。

## 设计理念

ENIAC Short Code 的核心创新是抽象：用符号记号法代替繁琐的直接机器编程。

<Callout type="default" emoji="💡">
  ENIAC Short Code 使用二进制编码的十进制数来表示数字，需要程序员人工将指令编码到打孔卡上。
</Callout>

程序写在纸上，每条语句包含变量名、运算符和值。编译器（也就是人类）将这些翻译成 ENIAC 的机器语言。

## 代码示例

```shortcode
A = 5
B = 3
C = A + B
```

这个简单示例展示了 ENIAC Short Code 如何让程序员用更易理解的符号表示法（虽然仍需人工编译）来表达算术运算，而不是直接使用机器码。

## 影响与遗产

**灵感来源：** 数学符号法、人工计算方法

**影响语言：** 所有后续编程语言

ENIAC Short Code 确立了一种激进的思想：计算机可以通过符号记号法编程，而无需物理重新配置。无论后续的汇编语言、FORTRAN 还是 Python，都继承了这种抽象分离的概念——将程序员的意图（以符号形式）与机器的执行（二进制）分开。

这一原则成为了所有软件开发的基础理念。
