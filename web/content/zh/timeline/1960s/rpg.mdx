# RPG

> _一种用自己的方式表达业务逻辑的语言。_

## 概览

| | |
|---|---|
| **诞生时间** | 1964 |
| **创造者** | IBM |
| **范式** | 命令式、报表导向 |
| **类型系统** | 静态、定格式 |
| **平台** | IBM i (AS/400) |
| **当前版本** | RPG IV (ILE RPG) |
| **许可协议** | 专有（IBM） |

## 诞生背景

RPG（报表程序生成器）由 IBM 于 1964 年创建，旨在替代在穿孔卡上编写报表的繁琐过程。它专为商业环境中常见的数据处理任务设计：读取文件、执行计算、生成格式化报表。

这门语言使用与其穿孔卡起源相呼应的定格式语法。每一列都有特定的含义，使程序高度可读但也相当僵化。

与大多数试图成为通用语言的尝试不同，RPG 始终忠于其原始使命：业务数据处理。这种专注使其能够在细分领域存活并繁荣超过 60 年。

## 设计理念

<Callout type="default" emoji="💡">
  "RPG 被设计用于从顺序文件中生成业务报表，并以最少的程序员干预完成。"
</Callout>

RPG 的定义特征是其**基于周期**的执行模型：
- 自动处理页眉和详细行
- 内置文件处理
- 自动计算的字段
- 边缘情况的异常处理

## 代码示例

```rpg
     H
     D CustName        S             35A
     D Balance         S              9P 2
     D Interest         S              9P 2
     D NewBalance       S              9P 2
     C                   READ        CUSTOMER
     C     *INLR         DOWEQ      *OFF
     C                   IF          Balance > 1000
     C     Balance       MULT       0.15       Interest
     C     Balance +     ADD        Interest   NewBalance
     C                   ENDIF
     C                   READ        CUSTOMER
     C                   ENDDO
```

## 影响与遗产

**灵感来源：** 穿孔卡处理、早期报表生成器

**影响语言：** RPG IV、CL（控制语言）

RPG 是历史上最成功的细分语言之一。虽然它从未走出 IBM 生态系统，但继续为关键业务应用提供动力：
- 许多中型企业仍在运行 1980 年代编写的 RPG 程序
- 现代 RPG IV 完全集成现代 API 和 Web 服务
- IBM i 平台在特定行业仍然受欢迎

RPG 的启示：一种语言不需要成为通用语言才能持久。在一件事上做到最好，可以胜过在很多事上做得还行。
