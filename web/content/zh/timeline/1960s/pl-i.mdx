# PL/I

> _IBM 的大胆尝试：创造一种适用于所有场景的语言。_

## 概览

| | |
|---|---|
| **诞生时间** | 1964 |
| **创造者** | IBM |
| **范式** | 命令式、过程式、面向对象（后期） |
| **类型系统** | 静态、强类型 |
| **平台** | IBM 大型机 |
| **当前版本** | PL/I for z/OS |
| **许可协议** | 专有（IBM） |

## 诞生背景

1960 年代初期，IBM 面临一个问题：他们有两个截然不同的计算环境。科学计算机使用 FORTRAN 进行数学运算，而商业计算机使用 COBOL 进行数据处理。维护两种语言既昂贵又低效。

IBM 决定创建一种统一的语言，能够同时适用于科学和商业应用。PL/I（编程语言一号）就这样诞生了。

尽管雄心勃勃，PL/I 从未获得 IBM 所期望的广泛采用。它主要仍是 IBM 大型机的语言，尽管它确实引入了许多创新功能。

## 设计理念

<Callout type="default" emoji="💡">
  "PL/I 被设计成为一种所有人的一切服务——科学、商业和系统编程的统一语言。"
</Callout>

关键特性：
- **异常处理**：首批具有结构化错误处理的主流语言
- **递归**：完全支持递归过程
- **指针**：直接内存寻址能力
- **多任务处理**：内置并发执行支持
- **字符串**：定长和变长字符串

## 代码示例

```pl1
/* 阶乘过程，带异常处理 */
FACTORIAL: PROCEDURE (N) RETURNS (FIXED BINARY(31));
    DCL N FIXED BINARY(31);
    DCL RESULT FIXED BINARY(31);
    ON ERROR BEGIN;
        PUT SKIP LIST('FACTORIAL 中发生错误');
        RETURN 1;
    END;

    IF N <= 1 THEN
        RESULT = 1;
    ELSE
        RESULT = N * FACTORIAL(N - 1);

    RETURN RESULT;
END FACTORIAL;
```

## 影响与遗产

**灵感来源：** FORTRAN、COBOL、ALGOL

**影响语言：** C（异常处理概念）、Ada（任务/异常模型）

PL/I 的异常处理模型影响了后来语言处理错误的方式。它的多任务处理功能超前于时代，但由于语言的复杂性而未被充分利用。

这门语言表明，试图让一种语言满足所有人的需求，往往会导致一种在各方面都不突出的语言。
