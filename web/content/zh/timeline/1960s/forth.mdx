# Forth

> _用逆波兰表示法说话的语言。_

## 概览

| | |

| **范式** | 栈式、连接式、自反射 |
| **类型系统** | 动态、无类型 |

## 诞生背景

Forth 由查尔斯·H·摩尔（Charles H. Moore）于 1968 年在麻省理工学院的射电天文台创建，当时他正在开发控制射电望远镜的软件。现有语言对于他需要的实时控制任务来说，要么太慢、要么占用内存太大、要么太不灵活。

摩尔的解决方案极为简单：彻底废除语法分析器。Forth 使用的是一个由"词"（word）组成的字典。每个词都是一个操作栈的过程。要相加两个数，只需输入 `+`，它会弹出栈顶的两个值，相加后把结果压入栈中。

这种连接式方法——程序结构来自词的组合而非语法——被证明对嵌入式系统极其强大。整个 Forth 环境可以塞进几千字节的内存中，非常适合资源严重受限的设备。

摩尔后来与他人共同创立了 FORTH, Inc.（1973 年），将 Forth 引入嵌入式系统、太空应用，甚至 DEC POP-11 小型计算机。

## 设计理念

Forth 的设计建立在三个核心概念之上：

1. **栈**：所有操作都通过向栈压入值和从栈弹出值来完成。由于位置决定顺序，因此不需要运算符优先级或括号。

2. **逆波兰表示法（RPN）**：`3 4 +` 意味着"压入 3，压入 4，相加"——运算符位于操作数之后。

3. **词（Word）**：Forth 中一切都是词（过程）。新词通过现有词来定义，为问题域创建词汇表。

<Callout type="default" emoji="💡">
  在 Forth 中，程序本身就是语法。除了"读取词、执行词"之外没有其他语法规则。
</Callout>

Forth 还开创了**自反射**——程序在运行时检查和修改自身结构的能力。这使其在实时控制方面具有独特的灵活性。

## 代码示例

```forth
\ 定义一个计算阶乘的词
: FACTORIAL ( n -- n! )
  DUP 0 = IF DROP 1 EXIT THEN
  DUP 1 - FACTORIAL *
;

\ 更习惯的递归版本
: FACT ( n )
  1 SWAP ( 从累加器=1 开始 )
  BEGIN DUP 0 >
  WHILE OVER * SWAP 1 - REPEAT
  DROP
;

\ 定义复合词
: SQUARE ( n -- n^2 ) DUP * ;
: CUBE ( n -- n^3 ) DUP DUP * * ;

\ 栈操作示例
5 SQUARE          \ 结果: 25
3 4 + 5 *         \ 结果: 35 (3+4=7, 7*5=35)
10 20 SWAP        \ 结果: 20 10 (交换栈顶两值)
```

这个例子展示了 Forth 的栈式范式：`DUP` 复制栈顶值，`SWAP` 交换栈顶两个值，`DROP` 移除栈顶值。

## 影响与遗产

**灵感来源：** Burroughs B5000（与其"无语法分析器"理念相同）

**影响语言：** Joy、Factor、Cat、RPL（惠普计算器）

Forth 对编程的贡献：

1. **连接式编程**：Factor 和 Joy 等语言直接从 Forth 的组合方法演变而来
2. **惠普计算器**：惠普在其图形计算器中使用的 RPL 语言直接源自 Forth
3. **引导开发**：Forth 的极小占用使其成为引导其他系统的完美工具
4. **交互式开发**：Forth 开创了现代 REPL 提供的即时反馈循环
