# Español — 拉丁字母效率

西班牙语（Español）在 AI 辅助编程的 token 效率中处于中等位置。作为使用相对透明正字法的拉丁字母语言，西班牙语的 token 成本约为英语的 85-100%，同时在某些语言结构上具有优势。

## 西班牙语 Token 特征

**形态结构**

西班牙语是罗曼语族语言，具有丰富的屈折变化：

- 动词变位：hablo, hablas, habla, hablamos, habláis, hablan
- 性数一致：el problema, la solución, los problemas

这意味着西班牙语单词每个 token 可以编码比英语更多的语法信息，但分词器必须处理更多变体。

## Token 效率对比

| 语言 | Token 成本（相对）| 信息密度 | 拉丁字母 |
|------|----------------|---------|---------|
| 英语 | 100%（基准）| 中等 | 是 |
| 西班牙语 | 85-100% | 中等 | 是 |
| 法语 | 80-95% | 中等 | 是 |
| 德语 | 75-90% | 中高 | 是 |
| 中文 | 40-60% | 非常高 | 否 |

<Callout type="info">
  西班牙语 token 效率与其他主要罗曼语相似。差异在代码生成任务中最明显，因为英语技术术语占主导地位。
</Callout>

## 最佳实践

### 宜

- **使用正式第二人称 (usted)** — AI 上下文中语法中立
- **使用 Anglicismos** — 英语技术术语在西班牙语提示词中效果良好
- **保持句子简洁** — 西班牙语允许压缩表达
- **使用无主语结构** — "se requiere" (is required) token 高效

### 忌

- **过多变音符号** — á, é, í, ó, ú 可能增加 token 数量
- **过于正式的构造** — "le rogamos que tenga a bien" 浪费 token
- **不一致的代码切换** — 混合西班牙语和英语时保持一致

## 示例提示词

**高效（推荐）：**
```
Escribe un quicksort en Python
```
Token 数：~7 | 含义：用 Python 写快速排序

## 结论

西班牙语对于等效任务的 token 成本约为英语的 85-100%。虽然不如中文或日语高效，但西班牙语提供：
- 与技术术语的拉丁字母兼容性
- 对母语人士熟悉的语法
- 广泛的 AI 模型理解

对于西班牙语使用者，建议是：
- 在西班牙语提示词中接受英语技术术语
- 保持提示词简洁直接
- 在专业上下文中使用正式语气
