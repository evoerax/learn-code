# 中文 — 最高 Token 效率

中文在 AI 辅助编程中实现了最高的 token 效率。与等效语义的英文相比，中文提示词可以用显著更少的 token 表达相同的意图，成本约为英语的 40-60%。

## 为什么中文效率高

**字符型文字系统**

与字母语言不同，中文使用表意文字系统，每个汉字都是一个自包含的语义单元。这意味着：

- 无需分词的词边界决定
- 每个字符独立携带大量意义
- 合成词（成语、合成词）由构成字符逻辑组成

**Token 化优势**

现代 LLM 对中文和拉丁字母语言的分词方式不同：

```
英语："concatenation" → 1 token（13 个字符）
中文："连接" → 2-3 tokens（2 个字符）
```

**语义密度**

中文实现高信息密度的原因：
- 最小形态屈折变化（无动词变位，无复数形式）
- 由语义成分构成的合成词
- 四字格（成语）表达复杂思想

## Token 成本对比

| 任务 | 英语 Token | 中文 Token | 节省 |
|------|-----------|-----------|------|
| 解释一个函数 | 45 | 18-22 | ~55% |
| 描述一个 bug | 30 | 12-15 | ~50% |
| 请求代码重构 | 25 | 10-13 | ~52% |
| 请求解释 | 20 | 8-10 | ~50% |

<Callout type="info">
  Token 数量是近似值，因模型而异。在中文数据训练充分的模型（GPT-4、Claude 3、Gemini）中，中文节省最为显著。
</Callout>

## 中文提示词最佳实践

### 宜

- **使用简洁表达** — 中文自然地压缩意义
- **使用技术术语** — 大多数 AI 术语在各语言间一致
- **用标点组织结构** — 清晰的结构有助于分词
- **使用古典中文** — 四字格 token 效率高

### 忌

- **过多的礼貌标记** — 请、麻烦、谢谢 增加 token 但不增加意义
- **冗余修饰语** — 中文不需要经常用"非常"、"真的"
- **混合文字** — 避免在中文文本中不必要的英语单词

## 示例提示词

**高效（推荐）：**
```
用 Python 写个快速排序
```

**冗长（效率较低）：**
```
麻烦你帮我用 Python 编程语言写一个快速排序算法可以吗？谢谢
```

## 结论

中文是 AI 辅助编程中 token 效率最高的主要语言。通过正确的提示技术，中文比英语节省 40-60% 的 token 以获得相同结果。
