# English — 基准语言

English（英语）是 AI 辅助编程中测量 token 效率的基准语言。作为主要 LLM 中训练最充分的语言，英语提供了最广泛的技术术语覆盖和最可靠的 AI 理解，尽管与等效语义的中文相比，token 成本约为 2 倍。

## 为什么英语是基准

**最大的训练语料库**

英语主导 LLM 训练数据：

- 约 60-70% 的 Common Crawl 是英语
- 大多数文档、教程和技术写作都是英语
- AI 模型对英语细微差别的理解最深

**技术术语**

英语技术术语被所有 AI 模型普遍认可：

```
API, REST, CRUD, JSON, SQL, Git, HTTP, DNS, TCP/IP
```

这些术语 token 效率高（每个 1 token），无论提示词用什么语言都被普遍理解。

## Token 效率对比

| 语言 | Token 成本（相对）| 信息密度 | 拉丁字母 |
|------|----------------|---------|---------|
| 英语 | 100%（基准）| 中等 | 是 |
| 西班牙语 | 85-100% | 中等 | 是 |
| 中文 | 40-60% | 非常高 | 否 |

## 最佳实践

### 宜

- **保持简洁** — "fix bug" 和 "please fix the bug" 效果一样
- **使用技术术语** — 普遍 token 高效
- **结构清晰** — 项目符号和编号列表减少歧义

### 忌

- **冗长礼貌** — "Could you possibly..." 增加 token 但无价值
- **歧义代词** — 明确 "it" 指代什么

## 混合策略

许多开发者成功使用中英文代码切换：

```
用 Python 写 quicksort，要求：
1. 时间复杂度 O(n log n)
2. 使用 list comprehension
```

这结合了中文的效率和英语的技术术语。

## 结论

英语是 AI 编程的基准，因其训练主导地位和技术术语覆盖。虽然 token 成本是中文的 2 倍，但英语对于：
- 最大化 AI 理解
- 代码生成准确性
- 技术精确性

仍然必不可少。最优策略通常是双语的：英语用于技术术语，中文用于结构和效率。
