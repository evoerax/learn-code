# 1990s — 互联网时代

1990 年代将编程从一门专业手艺转变为一个全球性行业。万维网的出现催生了对能够处理网络通信、动态内容和快速开发的语言的迫切需求。Python、Java、JavaScript、Ruby 和 PHP 在这个十年中涌现——从根本上改变了谁能编程、软件能做什么，以及构建软件的速度。

## 核心主题

**Web 改变了一切。** 在 1990 年代之前，编程针对的是单机或局域网。Web 引入了无状态协议、HTML 渲染、客户端-服务器架构，以及快速迭代的需求。PHP 字面意思就是"个人主页工具"。JavaScript 在浏览器中运行。Java applet 承诺提供丰富的 Web 内容。互联网让任何有浏览器和文本编辑器的人都能编程。

**Java 的大胆承诺。** "一次编写，到处运行"是杀手级的卖点。Java 编译为字节码，在 JVM 上运行，而 JVM 被移植到了每个平台。这解决了 1980 年代的部署噩梦——当时你必须为每个目标操作系统重新编译 C++。Java 成为企业后端、Android 应用和大型系统的默认选择。JVM 本身也成为一个平台，后来的语言（Scala、Clojure、Kotlin）都会以它为目标。

**Python 的悄然崛起。** Python 的设计理念很激进：代码应该是可读的。强制缩进、英语关键字和"电池内置"的标准库使其成为对初学者最友好、同时又足够强大的语言。世界花了二十年才跟上，但 Python 从 1991 年到今天的轨迹，讲述了一个押注开发者体验并最终获胜的语言的故事。

**Ruby 与开发者幸福感。** 松本行弘明确地将 Ruby 设计为让程序员快乐的语言。"我们常常关注机器的需求，"他说，"但我想关注人的需求。"这一哲学最终在 Ruby on Rails（2004）中达到顶峰，证明了一个为快乐而设计的语言也能为商业带来巨大的生产力。

**函数式编程进入主流。** Haskell 是第一个将纯函数式编程、惰性求值和强大的类型类系统结合起来的语言。虽然最初在工业界并未广泛采用，但 Haskell 的研究成果——monad、类型类、惰性求值——影响了随后出现的每一种主要语言。OCaml 将 ML 的类型推断带入了实用的、支持面向对象的语言。

## 展望未来

1990 年代结束时，互联网泡沫正处于全盛期。Web 已经证明了解释型、动态类型语言可以支撑全球性业务。2000 年代将以更务实的姿态回应：结合多种范式最佳思想的语言、针对特定平台（JVM、.NET）的语言，以及解决 Web 成功所带来的扩展挑战的语言。
