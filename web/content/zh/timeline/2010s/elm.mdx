## Origin Story

Evan Czaplicki 于2012年在普林斯顿大学作为论文项目创建了 Elm。与传统 JavaScript Web 开发不同，Elm 提出了一种完全不同的方法：一种编译为 JavaScript 的纯函数式语言，强调可靠性和可维护性。

Elm 的决定性时刻是它证明了 Web 应用程序实际上可以拥有零运行时错误。通过其严格的类型系统和纯函数，Elm 使整类 bug 不可能发生。像 NoRedInk 和 Puppet 这样的公司在采用 Elm 后报告了生产错误的显著减少。

Elm 架构 (TEA) — 通过 Model-Update-View 模式的单向数据流 — 之后影响了 Redux 和 JavaScript 生态系统中的许多其他状态管理库。

## Design Philosophy

Elm 极度简单。没有运行时异常，不需要 null 检查，没有类型转换陷阱。类型系统是可推断的，但在需要时是显式的。每个函数都是纯的，除非明确标记为"命令"或"订阅"。

架构强制执行严格的单向数据流：用户 → 消息 → 更新 → 模型 → 视图。这种可预测性使调试变得系统化而非神秘化。

<Callout type="default" emoji="💡">
  Elm 著名的"你不允许运行这段代码"错误消息开创了新一代有用的编译器错误消息，现在影响了 Rust、TypeScript 和许多其他语言。
</Callout>

## Code Example

```elm filename="Elm 示例.elm"
-- Elm 架构：Model, Update, View

-- Model: 应用程序的状态
type alias Model =
    { count : Int
    , message : String
    }

initialModel : Model
initialModel =
    { count = 0
    , message = "点击增加计数"
    }

-- Messages: 触发更新的用户操作
type Msg
    = Increment
    | Decrement
    | Reset

-- Update: 基于消息转换模型的纯函数
update : Msg -> Model -> Model
update msg model =
    case msg of
        Increment ->
            { model | count = model.count + 1 }
        
        Decrement ->
            { model | count = model.count - 1 }
        
        Reset ->
            { model | count = 0 }

-- View: 声明式 HTML 渲染
view : Model -> Html Msg
view model =
    div []
        [ button [ onClick Decrement ] [ text "-" ]
        , span [] [ text (String.fromInt model.count) ]
        , button [ onClick Increment ] [ text "+" ]
        , p [] [ text model.message ]
        ]
```
