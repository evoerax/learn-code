## Origin Story

Graydon Hoare 于 2006 年在 Mozilla Research 工作时创建了 Rust，以 "rustust" 真菌命名——象征其能在恶劣条件下生存的能力。Mozilla 于 2009 年正式开始赞助该项目，首个稳定版本于 2010 年发布。

Rust 诞生于一个实际问题：如何在没有 C 和 C++ 中那些内存 bug 的情况下编写底层系统代码。悬空指针、缓冲区溢出和数据竞争等经典问题曾让整个行业损失数十亿美元用于修复安全漏洞和崩溃。

突破在于所有权系统——一组编译器在编译时强制执行的规则，在没有垃圾回收的情况下消除整类 bug。程序可以获得确定性的性能（无 GC 停顿），同时保持内存安全。

## Design Philosophy

<Callout type="default" emoji="💡">
  "Rust 的目标是给予程序员所需的底层控制能力，同时避免底层控制历来引入的 bug。" — Graydon Hoare
</Callout>

Rust 的核心创新是所有权和借用系统：

- **所有权**：每个值有且只有一个所有者；当所有者离开作用域时，值被释放
- **借用**：你可以引用值而不取得所有权，编译器强制执行"多个不可变引用或一个可变引用"的规则（两者不能同时存在）
- **生命周期注解**：编译器追踪引用的有效期限，防止悬空指针

这意味着 Rust 可以在编译时捕获使用后释放、双重释放和数据竞争——在程序运行之前。

## Code Example

```rust filename="rust_ownership.rs"
// Rust 的所有权系统在编译时防止内存 bug

fn main() {
    // 所有权转移（移动语义）
    let s1 = String::from("hello");
    let s2 = s1; // s1 被移动到 s2，s1 不再有效
    // println!("{}", s1); // 错误：移动后借用值
    
    // 借用（引用）
    let s3 = String::from("world");
    let len = calculate_length(&s3); // 传递引用而非所有权
    println!("{} 有 {} 个字符", s3, len); // s3 仍然有效
    
    // 可变借用
    let mut counter = 0;
    let inc = &mut counter;
    *inc += 1;
    // 当 inc 存活时不能有其他可变引用
    
    // 带所有权的模式匹配
    let data = Some(vec![1, 2, 3]);
    match data {
        Some(vec) if vec.len() > 2 => println!("获取了 {} 项", vec.len()),
        _ => println!("数据不足"),
    }
}

fn calculate_length(s: &String) -> usize {
    s.len() // 借用 s，不取得所有权
}
```
