## 诞生背景

LISP（列表处理）由约翰·麦卡锡于 1958 年在 MIT 创建。它源于麦卡锡关于用代数函数描述计算的理论论文。与专为数值效率设计的 FORTRAN 不同，LISP 是为符号计算而设计的——处理符号和列表而非数字。

最初的实现由史蒂夫·罗素完成，他将麦卡锡的理论论文翻译成了 IBM 704 上的可运行代码。这使 LISP 成为继 FORTRAN 之后第二门拥有实现的高级语言。

LISP 成为人工智能研究领域数十年的首选语言。其符号处理能力使其非常适合问题求解、定理证明和早期 AI 研究。

## 设计理念

<Callout type="default" emoji="💡">
  "最强大的编程语言不是拥有最多特性的语言，而是能最优雅地表达任何计算的语言。"——约翰·麦卡锡
</Callout>

LISP 引入了数十年后オ被广泛采用的激进理念：

- **代码即数据**：程序被表示为可被其他程序操作的列表
- **同像性**：代码的结构与其处理的数据结构相同
- **一切皆列表**：甚至函数和运算符也表示为列表
- **垃圾回收**：自动内存管理在 LISP 中率先实现

标志性的括号语法虽然有时被批评为"过度"，但它实现了语言强大的宏系统和反射能力。

## 代码示例

```lisp
;; LISP 著名的括号语法
(defun factorial (n)
  (if (<= n 1)
      1
      (* n (factorial (- n 1)))))

;; 代码即数据 - LISP 可以操作自己的代码
(quote (+ 1 2))  ; 返回列表 (+ 1 2)
(eval (quote (+ 1 2)))  ; 返回 3

;; 函数式风格
(mapcar #'factorial '(1 2 3 4 5))  ; (1 2 6 24 120)
```

## 影响与遗产

**灵感来源：** Lambda 演算、ENIAC Short Code

**影响语言：** Scheme、Racket、Clojure、Haskell、Python、Ruby、JavaScript、Julia

LISP 对编程的影响是深远而普遍的：

1. **条件表达式**（if-then-else）是为 LISP 发明的，后被所有语言采用
2. **递归**作为一种编程技术通过 LISP 得到推广
3. **垃圾回收**在 LISP 中率先实现，现已在 Java、Python、Go 等语言中使用
4. **函数式编程**概念如 map、reduce 和一等函数来自 LISP
5. **树数据结构**和符号表达式解析是 LISP 的创新

从 Python 到 JavaScript 再到 Ruby，每门动态类型语言都继承了 LISP 的遗产——将函数视为一等对象，强调表达力而非语法糖。
