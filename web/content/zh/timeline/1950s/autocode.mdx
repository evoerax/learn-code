# Autocode

> _证明高级语言可行的先驱者。_

## 概览

| | |
|---|---|
| **诞生时间** | 1952 |
| **创始人** | 艾利克·格伦尼（曼彻斯特大学） |
| **范式** | 命令式、编译型 |
| **类型系统** | 静态、弱类型 |

## 诞生背景

Autocode 由艾利克·格伦尼（Alick Glennie）于 1952 年在曼彻斯特大学创建，是世界上最早的高级编程语言之一。它专门为曼彻斯特 Mark 1 计算机设计——世界上最早的存储程序计算机之一。

在 Autocode 出现之前，程序员必须用机器码或汇编语言编写程序，这既繁琐、容易出错，还需要深入了解硬件。格伦尼试图创建一种语言，让程序员能够用更自然、更接近数学的方式表达算法，同时仍能生成高效的代码。

"Autocode"这个名字是字面意义上的：它是一种能够自动生成代码的代码，通过编译器实现。

## 设计理念

<Callout type="default" emoji="💡">
  Autocode 引入了一个革命性的概念：编译器可以将高级数学 notation 自动翻译成高效的机器码，为所有后续编程语言铺平了道路。
</Callout>

Autocode 的关键创新：
- 类似数学 notation 的代数语法
- 使用子程序调用实现代码重用
- 支持数组（称为"向量"）
- 自动寄存器分配
- 最早使用编译器而非解释器的语言之一

## 代码示例

```autocode
 PROGRAM FACTORIAL
 INTEGER N, F
 READ N
 F := 1
 FOR I := 1 TO N DO
   F := F * I
 ENDFOR
 PRINT F
 END
```

这个阶乘程序展示了 Autocode 的代数语法和控制结构。赋值运算符 := 和数学 notation 的使用影响了后来的语言设计。

## 影响与遗产

**灵感来源：** 机器码、数学 notation

**影响语言：** CPL、ALGOL 58、FORTRAN

Autocode 的意义在于它的概念验证：它证明了高级语言在实践中是可行的。它的代数语法直接影响了 ALGOL 58，进而影响了 Pascal、C，最终影响了几乎所有现代编程语言。

编译器的概念——将人类可读的代码翻译成机器码——通过 Autocode 的成功得以巩固。如果没有 Autocode，FORTRAN 和 ALGOL 的发展可能会大大延迟。
