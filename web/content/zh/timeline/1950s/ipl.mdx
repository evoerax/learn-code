# IPL（信息处理语言）

> _让计算机学会"思考"符号的语言。_

## 一览

| | |

## 起源故事

IPL（Information Processing Language，信息处理语言）诞生于1956年，由兰德公司的三位科学家 Allen Newell、Herbert Simon 和 Cliff Shaw 共同创造。这三位研究者当时正从事人工智能研究，他们需要一种能够处理符号信息——而非仅仅处理数字——的编程语言。

这个项目的诞生源于 Simon 和 Newell 在定理证明和认知心理学方面的研究。他们希望构建能够模拟人类问题解决过程的程序，这就要求程序能够表示和操作符号，而不仅仅是进行算术运算。正是这一需求，使 IPL 成为世界上第一门人工智能编程语言。

"信息处理语言"这个名称本身就体现了它的核心目的：以符号方式处理信息，而非数值计算。这一定位使其与同时代专注于数学计算的 FORTRAN 等语言形成了鲜明对比。

## 设计理念

<Callout type="default" emoji="💡">
  IPL 的设计初衷不是计算答案，而是以符号形式表示知识和推理过程。
</Callout>

IPL 开创性地引入了以下概念：

- **链表结构**：IPL 最早引入了链表（linked list）这一数据结构来组织数据，这一结构至今仍是编程的基础
- **符号处理**：程序能够操作代表概念的符号，而不仅仅是数值
- **递归函数**：IPL 支持递归，允许函数调用自身
- **符号操作**：创建、修改和遍历符号表达式的能力
- **基础 AI 操作**：内置关联查找（association）和搜索等操作

IPL 的设计直接影响了 LISP 的诞生，尽管 LISP 在 lambda 演算的基础上将这些理念发挥得更加淋漓尽致。

## 代码示例

```lisp
; IPL 风格的符号列表处理
; 列表被表示为一系列元素的序列

; 定义一个符号列表
(LIST (A B C D))

; 关联表查找——通过键找值
(ASSOC 'KEY1 '((KEY1 . VALUE1) (KEY2 . VALUE2)))

; 递归遍历
(DEFINE (MEMBER X LST)
  (IF (NULL LST)
      NIL
      (IF (EQ X (CAR LST))
          T
          (MEMBER X (CDR LST)))))
```

## 影响与遗产

**灵感来源：** 无（开山之作）

**影响了：** LISP、POP-2、Python、Ruby

IPL 的遗产主要体现在历史意义和深远影响上：

1. **首门 AI 语言**：它开创性地证明了计算机可以操作符号、表示知识
2. **列表处理基础**：链表和列表操作的概念由 IPL 首创，成为 LISP 的核心
3. **递归编程**：IPL 展示了递归作为一种实用编程技术的可行性
4. **人工智能作为学科**：用 IPL 编写的程序（定理证明器、问题求解器）帮助确立了人工智能作为独立学科的地位

尽管 LISP 最终在流行度上超越了 IPL，但 IPL 是概念验证，证明了符号 AI 编程的可行性。如果没有 IPL 的开创性工作，LISP 可能永远不会诞生。
