# 汇编语言

> _机器的语言，一次一条指令。_

## 概览

| | |
|---|---|
| **年份** | 1947 |
| **创造者** | Kathleen Booth（伦敦大学） |
| **范式** | 命令式、底层 |
| **类型系统** | 无（原始内存） |
| **平台** | CPU特定 |
| **当前版本** | 因架构而异（x86-64, ARM, RISC-V） |
| **许可证** | N/A（公共领域 / CPU厂商） |

## 起源故事

汇编语言由 Kathleen Booth 于 1947 年在伦敦大学工作时为 ARC（自动继电器计算机）发明。Booth 没有直接编写二进制机器码，而是创造了一种将每条指令映射到助记符的符号语言——例如用"ADD"代替二进制操作码。

关键创新是"汇编器"：一个将这些符号指令翻译成原始机器码的程序。这是使编程超越数学家记忆二进制操作码的第一步。

在汇编出现之前，编程意味着拨动开关或连接插板。汇编出现后，程序员可以用可读的符号表达逻辑，同时保持对硬件的完全控制。

## 设计哲学

<Callout type="default" emoji="💡">
  "汇编不是一门需要学习的语言——而是一种需要理解的必需品。"
</Callout>

汇编体现了几个核心原则：

- **一对一映射**：每条汇编指令恰好翻译成一条CPU指令
- **无抽象**：每个寄存器、内存地址和硬件细节都是显式的
- **最高性能**：代码和硬件之间零开销
- **完全控制**：直接访问CPU特性如标志位、栈、中断

没有编译器优化你的代码——程序员就是优化器。每条指令都完全按照编写的方式执行。

## 代码示例

```asm filename="loop.x86"
section .data
    msg db '你好，世界！', 0
    len equ $ - msg

section .text
    global _start

_start:
    ; 写入标准输出
    mov eax, 4          ; sys_write 系统调用
    mov ebx, 1          ; 文件描述符（标准输出）
    mov ecx, msg        ; 消息指针
    mov edx, len        ; 消息长度
    int 0x80            ; 调用内核

    ; 退出，状态码 0
    mov eax, 1           ; sys_exit 系统调用
    xor ebx, ebx         ; 状态码 = 0
    int 0x80             ; 调用内核
```

## 影响与遗产

**启发自：** 机器码、二进制记法

**影响：** C、C++、Rust、所有高级语言

汇编写是纯硬件和人类可读代码之间的桥梁。每种处理器架构都有自己特有的汇编语言（x86、ARM、RISC-V、MIPS），理解汇编能让人深入了解计算机的实际工作原理。

尽管今天直接编写汇编的人很少，但它仍然是必不可少的：
- 编写引导加载程序和固件
- 优化关键性能瓶颈
- 理解编译器如何工作
- 安全研究和恶意软件分析

C 和 Rust 等现代语言最终编译为汇编，使得汇编知识对调试和优化非常有价值。
