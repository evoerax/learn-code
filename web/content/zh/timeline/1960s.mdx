# 1960s — 结构与抽象

1960 年代面临"软件危机"——程序变得如此复杂，以至于无法可靠地构建和维护。解决方案是结构化：结构化编程、结构化数据和结构化思维。这个十年发明了子程序、代码块、对象和交互式终端。

## 背景

| | |
|---|---|
| **时代特征** | 结构化编程、分时系统、软件危机 |
| **代表硬件** | IBM System/360、PDP-1、GE-265 |
| **里程碑** | NATO 软件工程会议（1968）创造了"软件工程"一词 |
| **核心挑战** | 随着程序从数百行增长到数千行，管理日益增加的复杂性 |

| SNOBOL | 1962 | 贝尔实验室 | 模式匹配和字符串操作 |
| PL/I | 1964 | IBM & SHARE | 尝试统一科学与商业的语言 |

## 核心主题

**ALGOL 的深远影响。** ALGOL 60 在商业上从未被广泛使用，但它是有史以来最具影响力的语言设计。块结构（begin/end）、词法作用域、递归和形式语言语法（巴科斯-诺尔范式）都来自 ALGOL。C、Pascal、Java、Python——几乎所有现代语言都可以追溯其语法到 ALGOL。

**对象的诞生。** Simula 被创建用来模拟物理系统——港口中的船舶、气体中的分子。解决方案：拥有状态和行为、组织成类的对象。Simula 67 给了我们类、对象、继承和虚方法。Smalltalk 和 C++ 后来完善了这些思想，但 Simula 发明了它们。

**让所有人都能计算。** BASIC 在当时是革命性的：一种简单到没有 CS 背景的大学生也能学习的语言。达特茅斯的分时系统让几十名学生可以同时使用一台计算机。BASIC 为个人计算播下了种子——十年后，它将随每台微型计算机一起发货。

**系统编程分道扬镳。** BCPL 和 B 去掉了类型和抽象，以贴近硬件。这种思路直接导向了 C 和 Unix——一种刻意的选择，认为性能和控制比安全性更重要。

## 展望未来

1960 年代留下了两个重大问题：如何将软件扩展到数百万行代码？以及如何让对象成为*主要*抽象，而不仅仅是模拟工具？1970 年代将用结构化编程方法论回答第一个问题。1980 年代将用面向对象革命回答第二个问题。
