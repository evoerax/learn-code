# 时间线总览

> 编程语言的历史，是思想的历史。每个十年都带来了新的问题，也催生了将全新思维方式凝结成形的新语言。

本时间线涵盖**八个十年**中的 **100+ 种编程语言**，按语言诞生的年代及其所要解决的问题加以组织。

## 跨越年代的模式

驱动语言演化的几对核心张力：

**安全性 vs. 性能** — 从 C 语言的"信任程序员"哲学，到 Rust 的所有权系统，语言始终在控制力与正确性之间寻求平衡。

**简洁性 vs. 表达力** — BASIC 以简洁为设计目标，APL 则追求极致表达力。大多数语言试图找到中间地带——而在哪里，各有主张。

**静态类型 vs. 动态类型** — FORTRAN 是静态类型的，LISP 是动态的。两种传统至今都在蓬勃发展，TypeScript 和 mypy 等语言则尝试融合两者的优势。

**通用语言 vs. 领域专用语言** — SQL、MATLAB 和 PostScript 在特定领域表现出色；C、Python 和 Java 则试图解决一切问题。

理解这些张力，会让这段历史远比一串日期和名字更加丰富。
