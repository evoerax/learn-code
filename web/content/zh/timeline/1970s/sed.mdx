## 起源故事

sed 诞生于 1974 年的贝尔实验室，是 grep 编辑器的自然进化。Lee E. McMahon 意识到程序员经常需要对多个文件进行重复的文本转换——替换、删除、插入——而手动使用 ed 既繁琐又容易出错。

名字代表 "stream editor"（流编辑器）。与在内存中打开完整文件的交互式编辑器不同，sed 逐流处理文本，非常适合 Unix 的小而可组合工具哲学。管道现在可以包含复杂的文本操作，而无需离开 shell。

McMahon 的设计受到了早期 ed 编辑器的影响，但 sed 添加了关键功能：脚本驱动的非交互操作、正则表达式支持，以及保持缓冲区（用于多行操作的临时工作空间）。

## 设计哲学

<Callout type="default" emoji="💡">
  "sed 是一个服从命令的仆人。给它精确的指令，它就会完美地执行任何数量的文本转换。" — Unix 哲学的应用
</Callout>

sed 的核心概念：

- **行导向**：sed 逐行处理文件，顺序应用转换
- **正则表达式**：使用基本正则表达式（BRE），支持分组、交替和量词
- **原地编辑**：`-i` 标志直接修改文件（后来的添加）
- **两个缓冲区**：模式空间（当前行）和保持缓冲区（跨行操作）
- **可寻址命令**：使用行号或模式仅对特定行应用转换

语法遵循形式：`sed [选项] '地址 命令' 文件`

## 代码示例

```sed
# 注释以 # 开头
# 基本替换：s/模式/替换/标志
# g = 全局（每行所有匹配项）

# 全局将 'foo' 替换为 'bar'
sed 's/foo/bar/g' input.txt

# 仅对包含 'error' 的行进行替换
sed '/error/s/warning/critical/' log.txt

# 删除第 1-10 行
sed '1,10d' file.txt

# 在第 3 行前插入文本
sed '3i\新行内容' file.txt

# 在第 5 行后追加文本
sed '5a\追加的内容' file.txt

# 使用 & 引用匹配的文本
sed 's/[0-9]+/(&)/' numbers.txt

# 使用保持缓冲区的多行操作
# 反转文件中的行（简化版）
sed '1!G;h;$!d' file.txt
```

```bash
# 管道示例：从日志中提取 IP，排序，统计唯一值
cat access.log | sed 's/.*from \(.*\)\.*/\1/' | sort | uniq -c | sort -rn
```

## 影响与遗产

**启发自：** ed（Unix 行编辑器）、grep

**影响：** awk（扩展了 sed 的能力，添加条件-动作对）、Perl（文本处理）、现代 CLI 工具如 sd

sed 的持久贡献是证明了行导向的文本处理可以既强大又可组合。它建立了 Perl、Python 的 `re` 模块以及无数其他工具遵循的模式：读取文本，应用转换，输出结果。

`/pattern/action` 地址语法影响了我们对选择性文本处理的理解。每次你运行一个一行命令来转换 JSON、修复导入或匿名化日志时，你都在使用 sed 后代的思维。
