## Origin Story

Groovy由James Strachan于2003年在BEA Systems工作时创建。他对Java的冗长感到沮丧，希望创造一种结合Python、Ruby和Smalltalk动态特性与JVM强大功能和可移植性的语言。Groovy的第一个版本于2004年发布，迅速因其简洁的语法和与Java的无缝互操作性而获得关注。

2007年，Groovy被纳入Java社区流程（JSR-241），正式成为Java平台的标准脚本语言。2009年，红帽收购了拥有Groovy的SpringSource，确保了该语言的持续开发和商业支持。

## Design Philosophy

Groovy的设计原则是**"减少样板代码同时保持Java兼容性"**。每个语法特性都旨在消除Java所需的仪式性代码。语言采用"合理默认值"原则——你可以省略分号、返回类型、类型声明和括号，让Groovy推断你的意图。

Groovy还开创了可选类型——你可以编写动态类型代码进行快速原型开发，或添加静态类型注解以提高性能关键的生产代码性能。这种灵活性让开发者能够为每个场景选择合适的安全级别和生产力。

## Code Example

```groovy filename="groovy_example.groovy"
// 动态类型和闭包
def list = [1, 2, 3, 4, 5]
def doubled = list.collect { it * 2 }
println doubled // [2, 4, 6, 8, 10]

// 字符串插值和GString
def name = "World"
def greeting = "Hello, ${name}!"
println greeting // Hello, World!

// 更轻松的Builder模式
def xml = """
<html>
    <body>
        <p>${greeting}</p>
    </body>
</html>
"""

// 安全导航操作符
class User {
    String name
    Address address
}
Address addr = user?.address // 如果user为null则返回null而不是NPE

// 运行在JVM上，与Java无缝互操作
String javaString = "I am Java"
println javaString.toUpperCase()
```

## When to Use

**Good fit:**
- Gradle构建脚本（Android行业标准）
- Jenkins Pipeline自动化
- JVM上的快速原型开发
- 为Java应用添加动态脚本能力
- 领域特定语言（Groovy的Builder语法无与伦比）

**Not a good fit:**
- Android应用开发（首选Kotlin）
- 需要静态类型的高性能系统
- 偏好Java显式性的团队环境

<Callout type="info" emoji="🤔">
  Gradle是Android开发最流行的构建工具，使用Groovy作为其主要的领域特定语言（DSL）。虽然Gradle现在也支持Kotlin DSL，但Groovy仍然是原始的、广泛使用的选项。
</Callout>
