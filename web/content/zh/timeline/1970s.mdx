# 1970s — 系统与语义

1970 年代分裂为两个方向，至今仍定义着编程的面貌。一方是 C 和 Unix，拥抱底层控制和简洁性；另一方是 ML、Scheme 和 Smalltalk，探索类型理论、函数式编程和纯面向对象。这个十年给了我们构建操作系统的工具——以及理解程序含义的理论。

## 背景

| | |
|---|---|
| **时代特征** | Unix、结构化编程、类型理论 |
| **代表硬件** | PDP-11、VAX 11/780、早期微处理器 |
| **里程碑** | Unix 和 C 在贝尔实验室共同开发（1969–1973） |
| **核心挑战** | 构建可靠、可移植的系统软件 |

<Callout type="info">
  C 和 Unix 是共同开发的——彼此塑造对方。Unix 需要一种可移植的系统语言；C 需要一个真实的系统来编写。结果是一个操作系统，可以通过重写少量 C 核心代码移植到新硬件，而无需重新连线电路。
</Callout>

## 1970 年代的语言

| 语言 | 年份 | 创造者 | 意义 |
|------|------|--------|------|
| Pascal | 1970 | Niklaus Wirth | 简洁的结构化语言；标准教学语言长达 20 年 |
| C | 1972 | Dennis Ritchie（贝尔实验室） | 系统语言；Unix 实现；C++、Java、C# 的祖先 |
| Prolog | 1972 | Colmerauer & Roussel | 逻辑编程；声明式约束求解 |
| SQL | 1974 | Chamberlin & Boyce（IBM） | 关系数据库查询语言；至今仍占主导地位 |
| Scheme | 1975 | Steele & Sussman（MIT） | 具有词法作用域和尾调用优化的极简 LISP |
| Smalltalk | 1972 | Alan Kay（施乐 PARC） | 纯面向对象；一切皆对象；消息传递 |
| ML | 1973 | Robin Milner（爱丁堡） | 类型推断、模式匹配；Haskell、OCaml、F# 的祖先 |
| AWK | 1977 | Aho、Weinberger、Kernighan | 文本处理；以创造者命名 |
| sed | 1974 | Lee McMahon（贝尔实验室） | 流编辑器；正则表达式文本变换 |
| Modula-2 | 1978 | Niklaus Wirth | Pascal 继任者，支持模块和并发编程 |
| PostScript | 1976 | John Warnock | 页面描述语言；基于栈；驱动激光打印 |

## 核心主题

**C 的极简主义。** Dennis Ritchie 将 C 设计得足够小，程序员可以把整个语言装进脑子里。没有对象，没有垃圾回收，没有异常——只有控制流、函数和指针。这种简洁使 C 具有可移植性和速度，但同时也带来了危险：缓冲区溢出、悬空指针和未定义行为是 C 留下的遗产。

**类型理论进入实践。** ML 最初是作为定理证明的元语言开发的，但其类型系统是一个启示。Hindley-Milner 类型推断让编译器无需注释就能推导类型——你写代码，类型检查器找错误。参数化多态（泛型）在这里被形式化。ML 的类型系统影响了 1980 年以后构建的每一种静态类型语言。

**正确实现面向对象。** Alan Kay 在施乐 PARC 的 Smalltalk 是面向对象思想最纯粹的表达：一切皆对象，所有计算都是消息传递，运行时环境是一个活的、交互式的系统。Smalltalk 发明了现代 IDE，具有语法高亮、方法浏览器和集成调试器。

**数据查询变为声明式。** SQL 提出了一个革命性的问题：如果你描述*想要什么*数据，而不是*如何*获取它，会怎样？关系模型和 SQL 查询语言将应用逻辑与存储解耦——这种关注点分离被证明如此强大，至今仍主导数据管理。

## 展望未来

1970 年代给了程序员迄今最强大的工具。1980 年代将把面向对象编程推向主流，将其加入 C，并看着软件行业发生变革。
