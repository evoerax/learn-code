# DAO 治理机制

治理机制决定 DAO 如何做出决策和执行决策。机制选择显著影响参与度、公平性和有效性。

---

## 代币加权投票

最常见模式 — 一代币一票：

```solidity
contract Governor {
    struct Proposal {
        uint256 id;
        uint256 forVotes;
        uint256 againstVotes;
        bool executed;
    }
    
    function castVote(uint256 proposalId, uint8 support) external {
        // 0 = 反对, 1 = 支持, 2 = 弃权
        Receipt storage receipt = receipts[proposalId][msg.sender];
        require(!receipt.hasVoted, "already voted");
        
        receipt.hasVoted = true;
        receipt.support = support;
        receipt.votes = getVotes(msg.sender);
        
        if (support == 1) {
            proposals[proposalId].forVotes += receipt.votes;
        }
    }
}
```

## 代币加权投票的问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **鲸鱼主导** | 大持有者控制结果 | 中心化 |
| **低参与** | 冷漠、投票疲劳 | 5% 参与率常见 |
| **治理攻击** | 买多数，攻击协议 | 安全风险 |

## 替代投票机制

### 二次投票

投票成本二次方增加 — 减轻鲸鱼影响：

```
票数 = √(消耗代币数)

1 代币 → 1 票
100 代币 → 10 票
10,000 代币 → 100 票
```

### 信念投票

投票权重随时间累积：

```
信念 = 代币数 × 持有时间
```

长期信仰者有更多影响力。连续，而非离散。

## 委托

允许代币持有者委托投票：

```solidity
function delegate(address delegatee) external {
    require(delegatee != address(this), "cannot delegate to contract");
    
    delegates[msg.sender] = delegatee;
    _moveDelegation(delegates[msg.sender], delegatee, balanceOf(msg.sender));
}
```

## Governor Bravo (Compound)

最广泛分叉的治理合约：

| 阶段 | 描述 |
|------|------|
| 提议 | 创建提案 |
| 活跃 | 投票中 |
| 成功 | 达到法定人数 |
