## 区块链解决的问题

在 Bitcoin 之前，数字交易需要可信第三方（银行、支付处理商）来防止双花。区块链通过以下方式消除这种需求：

1. **分布式账本** — 副本存在于数千个节点
2. **共识规则** — 参与者就单一真相达成一致
3. **加密验证** — 任何人都可以验证交易有效性
4. **不可变性** — 一旦确认，交易无法逆转

## 区块链特性

| 特性 | 描述 |
|------|------|
| **去中心化** | 无单一控制点或故障点 |
| **透明性** | 所有交易公开可见 |
| **不可变性** | 已确认交易无法篡改 |
| **抗审查** | 无权威可以阻止或逆转交易 |
| **可编程性** | 智能合约实现链上自定义逻辑 |
