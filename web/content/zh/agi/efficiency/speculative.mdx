# 投机解码

加速 LLM 推理的解码方法。

---

## 核心思想

使用小模型快速生成候选，大模型验证。

```python
def speculative_decoding(model, draft_model, prompt, gamma=4):
    # 小模型快速生成
    draft_tokens = draft_model.generate(prompt, max_new_tokens=gamma)
    
    # 大模型并行验证
    full_tokens = prompt + draft_tokens
    logits = model(full_tokens)
    
    # 验证每个 token
    accepted = []
    for i, draft_tok in enumerate(draft_tokens):
        # 如果概率低于阈值，接受
        if logits[i+len(prompt)].argmax() == draft_tok:
            accepted.append(draft_tok)
        else:
            # 拒绝，重新采样
            break
    
    return accepted
```

## 加速比

| 场景 | 加速比 |
|------|--------|
| 简单任务 | 2-3x |
| 复杂推理 | 1.2-1.5x |
| 长序列 | 更高 |

---

## 时间线对比

```
无投机解码:
Token 1: 目标模型前向 (100ms)
Token 2: 目标模型前向 (100ms)
Token 3: 目标模型前向 (100ms)
...

有投机解码 (gamma=4):
Step 1: Draft x4 (25ms) + 目标验证 (100ms) ≈ 4 tokens
Step 2: Draft x4 (25ms) + 目标验证 (100ms) ≈ 4 tokens
...

接受率高时加速: ~2-3x
```

---

## 接受率影响因素

| 因素 | 影响 |
|------|------|
| **Draft/目标相似** | 接受率更高 |
| **高熵上下文** | 接受率更低 |
| **重复模式** | 接受率更高 |
| **gamma** | 权衡：更多计算 vs 并行性 |

---

## 条件

- Draft model 与大模型分布接近
- 硬件支持并行计算
- gamma 选择平衡延迟和接受率

---

## 实现

```python
# Medusa 风格：多个 draft heads
class MedusaModel(nn.Module):
    def __init__(self, base_model, num_heads=3):
        self.base = base_model
        self.draft_heads = nn.ModuleList([
            nn.Linear(hidden, vocab_size) for _ in range(num_heads)
        ])
```

Medusa 可以达到 2-3x 加速。
