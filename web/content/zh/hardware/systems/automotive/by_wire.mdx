# 线控与执行（三级）

线控（By-wire / x-by-wire / drive-by-wire）用电子信号链路替代了驾驶员输入与车辆执行器之间的传统机械或液压连接。驾驶员意图（转向角、制动力、加速踏板位置）由传感器捕获，转换为数字信号，由安全等级的控制器处理，再翻译为执行器指令。结果是一条全电子控制回路，主路径上无直接机械备份。

这与无人机 hobby ESC 方式有本质区别——飞控发出 PWM 或 DShot 信号直接驱动无刷电机驱动器。汽车线控增加了确定性、故障冗余和法规合规层，工程门槛高得多。

## 线控为何难

### 人因与安全关键性

线控系统失效可直接导致事故。与无人机飞行中丢失电机（可滑翔或开降落伞）不同，汽车在高速公路上失去转向或制动时驾驶员几乎无补救措施。这意味着每个线控系统必须按 ISO 26262 达到**汽车安全完整性等级（ASIL）**，其中 ASIL D 最严格（要求每组件每小时失效率 < 10⁻⁸）。

### 故障冗余

单点失效对主控制系统不可接受。汽车线控系统采用**冗余架构**，有两条或三条并行信号路径：

- **双冗余（2oo2）**：两条独立通道；一条失效后系统以性能降级状态继续运行。
- **三冗余（2oo3）**：三条通道；多数投票逻辑选出正确输出，单次失效后可维持全部性能。
- **Fail-safe（故障安全）**：至少要求系统检测到故障后进入安全状态（如施加弹簧回位制动、将电机扭矩降至零），而非允许不受控的加速或转向。

### 实时确定性

电子控制回路必须有界延迟。电动助力转向（EPS）中，电机助力须在驾驶员输入后 **2–10 ms** 内做出反应才感觉自然。制动线控中，从踏板行程到制动力完全施加须在 **100–150 ms** 内完成。这些是硬实时约束，非尽力而为。

### 环境严酷性

汽车执行器工作温度范围宽（车内 −40°C 至 +85°C，发动机舱 −40°C 至 +125°C），并伴随振动、湿度和来自点火系统及牵引电机逆变器的电磁干扰（EMI）。Arduino 驱动的舵机与量产汽车执行器之间的差距是巨大的。

## 四类线控系统

### 转向线控（Steer-by-wire, SbW）

转向线控完全取消了机械转向柱，替换为：

- **转向输入传感器**（非接触式，通常用磁阻式）读取方向盘角度和扭矩。
- 电子控制单元（ECU）将转向角映射为期望的前轮角度。
- 齿条或转向柱上的**电动助力转向（EPS）电机**驱动转向连杆。
- 方向盘内的**路感电机**向驾驶员提供人工反馈力（模拟轮胎力产生的转向阻力）。

量产转向线控罕见（日产和英菲尼迪在有限车型上提供过），因为法规安全要求极高。在车辆未达到极高可靠性阈值前，大多数法规仍要求保留机械备份转向柱。

### 制动线控（EBS / BBW）

电子制动系统使用**制动踏板行程传感器**和每个制动回路中的**压力传感器**：

- **电子制动力分配（EBD）** 根据载荷连续调节前后轴之间的制动力。
- **防抱死制动系统（ABS）** 调制每个车轮的制动压力，防止车轮抱死。
- **电子稳定控制（ESC）** 在个别车轮上有选择性地制动以修正不足转向或过度转向。

在**制动线控**（相对于传统 ABS）中，制动踏板与制动卡钳之间无液压连接。踏板传感器命令每个车轮上的制动电机或液压执行器独立工作。**再生制动**（电动车常见）通过同一电子系统与摩擦制动协调。

关键安全特性是**制动辅助**——如果驾驶员非常快速地踩下制动踏板（表明紧急情况），系统自动施加最大制动压力，无论驾驶员实际踩力多大。

### 油门线控（ETC）

电子油门控制（ETC）——也称 **drive-by-wire throttle**——从 1990 年代起在大多数汽油车上取代了机械油门拉线：

- 加速踏板有**双通道位置传感器**（两个独立电位计或霍尔效应传感器）。
- 发动机控制单元（ECU）读取踏板位置、发动机载荷、速度和其他信号，然后命令**节气门体电机**达到期望的进气流量。
- 节气门位置由节气门板上**冗余的节气门位置传感器（TPS）** 验证。
- 系统通过与轮速传感器闭环调节支持**怠速控制**、**巡航控制**和**牵引力控制**。

### 驻车线控（EPB）

电子驻车制动（EPB）使用执行器（通常为电机驱动卡钳或拉线拉动器）施加驻车制动，由仪表板按钮激活。现代 EPB 与稳定控制系统集成，提供 **auto-hold**（停止时维持制动压力）和**动态紧急制动**（行驶中驾驶员拉住并保持 EPB 开关时施加全部制动力）。

## 安全架构

### ISO 26262 与 ASIL

ISO 26262 是道路车辆功能安全标准，定义 ASIL A（最低）到 ASIL D（最高）的汽车安全完整性等级。转向和制动的线控系统通常达到 ASIL D——最高完整性等级，要求：

- **硬件故障容忍**：双冗余或三冗余硬件，能容忍单组件失效而不丧失功能。
- **诊断覆盖率**：危险硬件失效的诊断覆盖率 > 99%。
- **软件开发**：符合 Automotive SPICE 流程，软件组件间无干扰。
- **验证与确认**：包括硬件在环（HIL）、整车级测试和边角案例分析的广泛测试。

### Fail-operational 与 Fail-safe

- **Fail-operational（故障可运行）**：故障后系统以降低能力继续运行。要求转向线控在单次故障后仍保持驾驶员转向控制能力。
- **Fail-safe（故障安全）**：故障后系统进入安全状态（如零扭矩、施加驻车制动）。可接受用于某些油门和驻车线控应用。

### 看门狗与自检

汽车 ECU 对传感器、执行器和 CPU 本身运行**连续自检**。如果检测到故障，系统在实时截止时间内进入安全状态。电机驱动器包括**过流保护**、**过温关断**和**死区插入**以防止 H 桥配置中的穿通。

## 样车实践

对于大学车队项目、大学生方程式或 DIY 无人车实验，全面汽车级线控既不经济也无必要。典型做法：

- **电机驱动执行**：无刷直流电机通过皮带或减速箱驱动转向或制动机构。
- **转向**：柱扭矩传感器测量驾驶员努力值；一个无刷直流电机在转向齿条上或独立执行器推动横拉杆。Arduino 或 ESP32 产生 PWM 命令，辅助单片机读取冗余踏板传感器。
- **制动**：由直流电机或气动活塞驱动的制动主缸；制动压力由压力传感器测量并在闭环中反馈。部分车队使用比例阀控制的制动线控。
- **紧急停止**：一个硬件**死机开关**（蘑菇头按钮）可在任何软件状态下切断所有电机驱动器——这对任何可能造成伤害的车辆都是必须的。

这与量产线控的差距是巨大的——但样车平台教授相同的基本概念：传感器 → ECU → 执行器，含闭环反馈和故障安全逻辑。
