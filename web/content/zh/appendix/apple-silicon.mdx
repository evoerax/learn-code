# 苹果芯片进化史

苹果公司计算机产品线历经 **摩托罗拉 68k → IBM/摩托罗拉 PowerPC → Intel x86 → 自研 ARM（Apple Silicon）** 四次平台变迁；与此同时，**iPhone / iPad** 上的 **A 系列** SoC 从外购方案演进到与 Mac 共享同一套架构思想。下文以**时间线 + 设计转折**为主，不穷尽每一代 SKU。

若需对照 **x86 与 ARM 产业背景**，见 [x86 与 ARM](/zh/appendix/x86-vs-arm)。

---

## 四条时间线如何合在一起

| 阶段 | 大约年代 | 代表产品形态 | 核心特征 |
| :--- | :--- | :--- | :--- |
| **68k** | 1984–1990s 中 | Macintosh、早期 PowerBook | CISC、苹果依赖摩托罗拉供货 |
| **PowerPC** | 1994–2006 | Power Mac、iBook、iMac G3/G4 | **AIM 联盟** RISC，桌面性能与散热拉锯 |
| **Intel** | 2006–2020 | MacBook、Mac Pro、与 PC 同构 | x86 笔记本生态、Boot Camp、性能稳定 |
| **Apple Silicon** | 2020 至今 | Mac、iPad Pro、与 A 系列同哲学 | **AArch64**、高能效核、统一内存架构 |

移动侧 **A 系列**（2010 年起逐步加深自研）与 Mac **M 系列**（2020 起）在 ISA 层面同属 **ARMv8 / ARMv9** 家族，差别主要在 TDP、封装、I/O 与内存带宽规格。

---

## Motorola 68000 时代（Macintosh）

- **1984 — Macintosh 128K**：**Motorola 68000**，时钟约 8 MHz；图形界面与单机一体化设计，CPU 由摩托罗拉系谱迭代（68020、68030、68040）。  
- **意义**：苹果第一次把「个人电脑」定义在**一体化体验**上，但**指令集与供货方**仍与传统工作站/UNIX 圈迥异；后期 68k 在功耗与性能上与同期 x86/PowerPC 竞争吃力。

---

## PowerPC 时代（RISC 上桌）

- **1994**：**PowerPC 601** 进入 **Power Macintosh** —— Apple、IBM、Motorola **AIM 联盟** 推广的 **RISC** 路线。  
- **1990s–2000s**：**G3、G4（AltiVec）** 进入消费级桌面与笔记本；**clock / GHz 宣传战**与架构效率成为营销话术一部分。  
- **2003 前后 — Power Mac G5**：IBM **PowerPC 970（G5）** 推向桌面，**功耗与散热**在笔记本侧成为硬约束，与英特尔移动 Core 路线形成反差。  
- **意义**：苹果第一次系统性押注 **非 x86 的生态位**；其结束也证明：**性能曲线 + 瓦特数 + 笔记本形态** 会倒逼换芯。

---

## Intel 时代（与行业对齐）

- **2006**：乔布斯宣布 **改用 Intel Core**；首批 **Intel Mac** 使用 **Core Duo** 等移动处理器。  
- **2006–2020**：Mac 产品与 x86 **软件生态、虚拟化、驱动模型** 对齐；**Thunderbolt、Retina、超薄本** 等体验迭代建立在此平台上。  
- **意义**：苹果获得**可预期供应链**与**行业通用栈**，代价是与 PC **差异化主要依赖系统与工业设计**，而非核心硅片。

---

## iPhone / iPad：A 系列移动 SoC

- **2007**：初代 iPhone 使用 **三星等供应的 ARM SoC**（公版内核 + 苹果深度定制系统栈）。  
- **2010 — A4**（iPhone 4）：广泛视为苹果 **加强 SoC 话语权** 的起点——**Intrinsity + PA Semi** 团队并入后能力显性化。  
- **2013 — A7**：业界首批大规模 **64 位 ARM（AArch64）** 手机 SoC 之一，推动应用与系统向 **ARM64** 迁移。  
- **2017ff. — 性能核 / 能效核分化**：**big.LITTLE 类编排**成熟，**GPU、NPU（Neural Engine）、ISP、视频编解码** 与 CPU 同die 协同。  
- **意义**：苹果在 **瓦特与电池** 约束下把 **单线程、能效、加速器耦合** 推到行业前沿，为日后 Mac 迁移**预演架构与工具链**。

---

## Apple Silicon：M 系列与 Mac 回迁 ARM

- **2020.11 — M1**（MacBook Air、13" Pro、Mac mini）：首款 **面向 Mac 的 5nm 级 SoC**，**高性能核 + 能效核**，**统一内存架构（UMA）**；同一内存池被 CPU / GPU / Neural Engine 访问，显著改变传统 NUMA/独显显存模型。  
- **2021 — M1 Pro / M1 Max、M1 Ultra**（封装级多芯片）：抬升 **内存带宽与接口数量**，服务专业创作负载。  
- **2022 — M2**：迭代能效与规模；覆盖更多入门与中端 Mac。  
- **2023 — M2 Ultra、M3 系列**：M3 系引入新一代 GPU 与光线追踪等图形特性（按苹果公开表述），继续按 **制程 + 微架构 + 封装** 节奏发布。  
- **2024 — M4**（随 iPad Pro 等首发）：体现 **世代更新与 NPU 侧重点** 的延续；具体规格以苹果技术规格页为准。  
- **意义**：苹果把 **移动时代演练过的芯片哲学**（能效、加速器、内存一体）完整搬到 **桌面与工作站**，并辅以 **Rosetta 2** 缓解 x86 二进制过渡期。

---

## 其它苹果硅片（点到为止）

- **S 系列 / W 系列**：Apple Watch 等 wearables 上的 **SiP / 低功耗** 路线，强调 **传感器融合与续航**，与 A/M 主线并行。  
- **耳机 / 音频配件**：**H 系列**等芯片侧重无线与音频管线，不等同于通用应用处理器。

---

## 小结：苹果换芯的共同逻辑

1. **体验目标优先**：能否支撑更薄、更静、更长续航或更高持续性能。  
2. **供应链与差异化**：当外购方案在 **功耗曲线或路线图** 上与产品形态冲突时，倾向 **加深自研**。  
3. **软件消化能力**：四次迁徙都依赖 **强系统团队**（Classic → OS X → Rosetta → 通用二进制 / ARM64 工具链）。

---

进阶关键词：**Rosetta 2**、**统一内存**、**性能控制器（P/E 核调度）**、**AMX / Neural Engine**、**片上系统安全性（Secure Enclave 相关）**。若要深入指令级细节，可结合 ARM 官方 **ARMv8/v9 手册**与苹果开发者文档中的 **ABI 与调用约定**。
