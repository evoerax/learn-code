# Positional encoding & RoPE

- **Why**: Attention is permutation-equivalent without explicit position information.  
- **RoPE**: Encodes relative position via rotary transforms on Q/K; context extension often pairs with **base changes, NTK scaling, YaRN**, etc.  
- **Alternatives**: Learned absolute positions, ALiBi, and schemes tied to **linear attention / RWKV** in some architectures.
