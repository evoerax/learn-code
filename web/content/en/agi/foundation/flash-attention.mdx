# FlashAttention

FlashAttention (Dao et al., 2022) computes attention exactly while being 2-4x faster through IO-awareness and tiling.

---

## The IO Problem

Standard attention materializes the N×N attention matrix (O(n²) memory), which is slow due to memory bandwidth.

FlashAttention processes in **tiles** that fit in SRAM, reducing memory reads/writes.

---

## Algorithm

FlashAttention processes in tiles/blocks that fit in SRAM, keeping attention matrix in fast memory:

```python
def flash_attention(Q, K, V, block_size=128):
    """
    Q, K, V: (seq_len, d_k)
    Processes in blocks to reduce HBM reads/writes.
    """
    seq_len = Q.shape[0]
    d_k = Q.shape[1]
    output = zeros(seq_len, d_k)

    # Block-wise online softmax accumulation
    for i in range(0, seq_len, block_size):
        Qi = Q[i:i+block_size]
        mi_max = full(block_size, -inf)
        Li = zeros(block_size)
        Oi = zeros(block_size, d_k)

        for j in range(0, seq_len, block_size):
            Kj = K[j:j+block_size]
            Vj = V[j:j+block_size]

            # Compute attention block
            Sij = Qi @ Kj.T / sqrt(d_k)
            mij = Sij.max(dim=-1).values

            # Online softmax update
            Pij = exp(Sij - mij[:, None])
            Pij_Vj = Pij @ Vj

            # Safe normalization
            Li_new = exp(mij - mi_max) * Li + Pij.sum(dim=-1)
            Oi = exp(mij - mi_max)[:, None] * Oi + Pij_Vj

            mi_max = max(mi_max, mij)

        output[i:i+block_size] = Oi / Li[:, None]

    return output
```

**Key insight:** Instead of materializing N×N matrix, accumulate statistics (max, sum) block-wise using online softmax.
