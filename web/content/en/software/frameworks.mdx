# Frameworks & libraries

**Scope:** software used to **train** and **research** models: tensor runtimes, automatic differentiation, optimizers, and distributed strategies (data/tensor/pipeline parallel, ZeRO-style sharding, etc.).

---

## Typical layers (conceptual)

1. **Kernel / compiler layer** — fused ops, custom CUDA/ROCm, graph capture, compiler stacks (e.g. XLA/Inductor-style paths) when surfaced through a framework.
2. **Core library** — eager + graph modes, mixed precision, checkpointing.
3. **Higher-level training** — trainers, schedulers, integration with data pipelines and experiment trackers.
