# Smart Contract Languages

Smart contracts are programs deployed on blockchains that execute automatically when conditions are met. Writing them requires specialized languages designed for:

- **Determinism** — same input always produces same output
- **Gas optimization** — every computation costs fees
- **Security** — bugs can result in permanent fund loss
- **Formal verification** — mathematical proof of correctness

## Evolution of smart contract languages

### 2013-2015: Birth
- Solidity created by Gavin Wood for Ethereum
- Serpent (Python-like) and Mutan (Go-like) also existed
- Many security vulnerabilities discovered

### 2016-2019: Maturation
- Solidity became dominant
- Vyper emerged for formally verifiable contracts
- Best practices and security patterns established

### 2020-present: Specialization
- Rust for performance-critical applications
- Move for resource-heavy ecosystems
- ZK languages (Cairo, Circom) for privacy scaling

## Gas optimization fundamentals

Smart contract execution costs gas. Optimization matters:

```solidity
// Unoptimized — reads storage twice
function getBalance(address user) public view returns (uint) {
    return balances[user]; // COLD read
}

// Called again
function getTotal() public view returns (uint) {
    return balances[msg.sender] + balances[other]; // COLD reads
}

// Optimized — cache in memory
function getBalance(address user) public view returns (uint) {
    uint balance = balances[user]; // COLD read, now in memory
    return balance;
}
```
