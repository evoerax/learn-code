## The problem blockchain solves

Before Bitcoin, digital transactions required trusted third parties (banks, payment processors) to prevent double-spending. Blockchain eliminates this need through:

1. **Distributed ledger** — copies exist across thousands of nodes
2. **Consensus rules** — participants agree on the single source of truth
3. **Cryptographic verification** — anyone can verify transaction validity
4. **Immutability** — once confirmed, transactions cannot be reversed

---

## Key concepts

| Concept | Description |
|---------|-------------|
| **Block** | A container of verified transactions, linked to the previous block |
| **Chain** | Sequential chain of blocks, forming the blockchain |
| **Node** | A computer running blockchain software, storing and validating the ledger |
| **Miner/Validator** | Participants who bundle transactions into blocks and secure the network |
| **Gas** | The fee paid to execute transactions or smart contracts |
| **Nonce** | A counter used to ensure transaction uniqueness |

---

## Blockchain properties

| Property | Description |
|----------|-------------|
| **Decentralization** | No single point of control or failure |
| **Transparency** | All transactions are publicly viewable |
| **Immutability** | Confirmed transactions cannot be altered |
| **Censorship resistance** | No authority can block or reverse transactions |
| **Programmability** | Smart contracts enable custom logic on-chain |
