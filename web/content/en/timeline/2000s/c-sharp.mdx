## Origin Story

In the late 1990s, Microsoft faced a strategic challenge. Java was gaining popularity with its "Write Once, Run Anywhere" promise, and Windows needed a modern language that could compete. In 2000, Anders Hejlsberg—creator of Turbo Pascal and Delphi—unveiled C# (pronounced "C sharp") as part of the .NET initiative.

The language was designed from scratch to be safe, modern, and directly supported by the Common Language Runtime (CLR). It borrowed the best ideas from C++, Java, and Delphi while introducing innovative features like properties, events, and LINQ.

## Design Philosophy

C# was designed to be a simple, modern, object-oriented language. Microsoft aimed to create something that combined the power of C++ with the simplicity of Visual Basic. The result was a language that emphasized type safety, garbage collection, and component-oriented development.

The evolution of C# has been remarkable—from a Java-like language in 1.0 to a feature-rich powerhouse. Each version added significant capabilities: generics (2.0), LINQ (3.0), async/await (5.0), records (9.0), and pattern matching (8.0+).

## When to Use

**Good fit:**
- Enterprise applications and web services
- Game development with Unity
- Cross-platform desktop apps with .NET MAUI
- Windows-specific applications (WPF, WinForms)

**Not a good fit:**
- Systems programming where you need manual memory control
- Very lightweight scripts where startup time matters
- Frontend web development (use TypeScript instead)
