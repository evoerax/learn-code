## Origin Story

Graydon Hoare at Mozilla created Rust in 2010, named after a species of fungus known for its robustness. The project emerged from Mozilla's need for a language that could provide memory safety without garbage collection—essential for the Servo browser engine project.

Rust's defining innovation is its ownership system, which enforces memory safety at compile time through a set of rules that the compiler checks. This eliminates entire categories of bugs (null dereferences, data races, memory leaks) while imposing zero runtime cost.

## Design Philosophy

Rust's core philosophy is "fearless concurrency"—safe systems programming without data races. The ownership system treats memory access as a resource, applying the same principles used for managing files and network connections.

Zero-cost abstractions are central: higher-level constructs compile down to efficient machine code, matching hand-written C. The language offers functional features like pattern matching, iterators, and closures without runtime overhead.

## Code Example

```rust filename="Rust Example"
// Ownership and borrowing
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

// Concurrency without data races
use std::thread;

let data = vec![1, 2, 3];
let handle = thread::spawn(move || {
    data.iter().map(|x| x * 2).collect::<Vec<_>>()
});
let result = handle.join().unwrap();

// Pattern matching
match value {
    Some(x) if x > 0 => println!("Positive: {}", x),
    Some(x) => println!("Non-positive: {}", x),
    None => println!("Nothing"),
}
```

## When to Use

**Good fit:**
- Systems programming requiring memory safety
- Performance-critical applications
- WebAssembly compilation
- CLI tools with performance requirements

**Not a good fit:**
- Rapid prototyping where compile times matter
- Teams new to systems programming
- Simple scripts (use Python or Ruby)

## Controversies

<Callout type="warning">
  Rust's learning curve is steep. The borrow checker, while brilliant, often rejects valid programs until you learn its mental model. This "fighting the compiler" phase can frustrate newcomers but pays off in bulletproof code.
</Callout>
