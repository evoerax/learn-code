## Origin Story

In 2003, Mitchel Resnick and his team at the MIT Media Lab—building on the legacy of Seymour Papert's Logo—created Scratch as a modern answer to the question: how do we make programming accessible to everyone?

The key innovation was "blocks"—colorful, snap-together visual elements that eliminated syntax errors entirely. Instead of typing code, children drag and drop blocks to create programs. The online community launched in 2007, enabling kids worldwide to share their projects.

Scratch was designed with "low floor, high ceiling, wide walls" philosophy: easy to start, capable of complex projects, and wide enough to support many types of creations. By 2023, over 100 million Scratch projects had been shared by users in more than 200 countries.

## Design Philosophy

Scratch eliminates frustration. No syntax errors means no debugging messages about missing semicolons or mismatched braces. Programs are created through direct manipulation—seeing results immediately reinforces the connection between actions and outcomes.

The social dimension matters equally. The Scratch website is a creative community where young people showcase projects, remix each other's work, and receive feedback. This mirrors how real programmers actually learn: through collaboration and iteration.

## Code Example

```scratch filename="Catch Game"
// Visual blocks represented in markdown
// [When Green Flag clicked]
// repeat until <score = 10>
//   go to (random position)
//   say [Hello!] for (1) seconds
//   change [score v] by (1)
// end
// say [You win!] for (2) seconds
```
