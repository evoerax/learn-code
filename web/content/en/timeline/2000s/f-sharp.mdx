## Origin Story

Don Syme at Microsoft Research created F# in 2005, bringing functional programming to the .NET platform. The language was inspired by OCaml but designed specifically for the .NET ecosystem, allowing full interoperability with other .NET languages like C# and VB.NET.

F# was originally an academic research project, but its practical utility quickly became apparent. It shipped as a first-class .NET language, benefiting from the entire .NET ecosystem while bringing functional programming concepts to a wider audience.

## Design Philosophy

F# embraces functional programming principles—immutability by default, expressions over statements, and powerful type inference—while remaining a practical language for real-world problems. It follows the "suave" principle: whatever style you're comfortable with, F# supports it.

The language excels at data processing and analytical programming. Its type providers enable reading from databases, APIs, and data files with compile-time safety. F# also pioneers in domains like financial modeling where correctness matters deeply.

## Code Example

```fsharp filename="F# Example"
// Immutable data structures
type Person = { Name: string; Age: int }

// Discriminated unions for domain modeling
type Result<'T> =
    | Success of 'T
    | Failure of string

// Pipelines and higher-order functions
let adults = people
    |> List.filter (fun p -> p.Age >= 18)
    |> List.sortBy (fun p -> p.Name)
    |> List.map (fun p -> p.Name)

// Async programming
let! data = async {
    let! html = http.AsyncGetString(url)
    return parse html
}
```

## When to Use

**Good fit:**
- Financial modeling and quantitative analysis
- Data processing and ETL pipelines
- Domain-driven design with discriminated unions
- Scripting with strong typing

**Not a good fit:**
- Systems programming requiring manual memory control
- Teams unfamiliar with functional concepts
- Quick one-off scripts where F# setup overhead isn't justified
