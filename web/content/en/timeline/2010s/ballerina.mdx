## Origin Story

Ballerina was created by WSO2, a company known for its enterprise integration middleware, and released in 2017. The team observed that despite decades of progress in programming languages, connecting services over networks still required boilerplate code, external libraries, and manual error handling.

The insight was that integration—connecting APIs, services, databases, and message queues—had become a first-class concern that traditional languages weren't designed for. Ballerina was built from the ground up with network primitives as language features, not afterthoughts.

The language combines the performance of compiled languages with the productivity of scripting languages, and its syntax is designed to be visually expressive of integration concepts like flows, endpoints, and connectors.

## Design Philosophy

<Callout type="default" emoji="💡">
  "Ballerina is a programming language that makes it easier to use, combine, and create network services." — ballerina.io
</Callout>

Ballerina's unique features for cloud-native development:

- **Network primitives as language features**: HTTP, SQL, gRPC, Kafka are built into the type system
- **Structural concurrency**: Fibers (lightweight threads) with sequence diagrams generated automatically
- **XML and JSON as primitives**: Native support for modern data formats
- **Visual execution**: The language can generate sequence diagrams from code showing exactly how services interact

This means you write integration logic at a higher level of abstraction, and the language handles connection management, timeouts, and error handling consistently.

## Code Example

```ballerina filename="ballerina_service.bal"
// Ballerina's network-aware syntax - HTTP service with automatic error handling

import ballerina/http;

service / on new http:Listener(8080) {
    
    // Resource functions handle HTTP requests
    resource function get users/[int id]() returns json|error {
        // Database query is a language-level operation
        json user = check db->query(`SELECT * FROM users WHERE id = ${id}`);
        return user;
    }
    
    resource function post users(@http:Payload UserInput input) returns json|error {
        // Validation is built into the type system
        if input.name.length() == 0 {
            return error("Name is required");
        }
        
        // Insert and return generated ID
        int id = check db->execute(`INSERT INTO users (name) VALUES (${input.name})`);
        return { id: id, name: input.name };
    }
}

// Service composition with sequence diagram generation
service /integration on new http:Listener(8081) {
    resource function get profile(int userId) returns json|error {
        // Parallel calls to multiple services
        user, repos = wait { getUser(userId), getRepos(userId) };
        
        return { user: user, repoCount: repos.length() };
    }
}
```
