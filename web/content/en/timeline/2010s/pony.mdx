## Origin Story

Pony was created by Sylvan Clebsch in 2014 and released in 2015. Clebsch previously worked on the actor model in other languages and wanted to build a language where concurrency bugs were impossible by design—not just harder to write.

The name "Pony" came from Clebsch's work in gaming, where he jokingly referred to his colleagues as "ponies" when they shipped features quickly. The language was built from scratch with the goal of combining the safety of actor-based concurrency with the performance of native code.

What makes Pony unique is that it has no data races because its type system tracks what capabilities each actor has to access data. The compiler can prove your program is safe before it runs.

## Design Philosophy

<Callout type="default" emoji="💡">
  "Pony is a programming language that is type-safe, race-safe, memory-managed, and fast." — ponyc documentation
</Callout>

Pony's breakthrough is the **capabilities security model**:

- Every object has a capability (read, write, share, etc.) that governs what operations are allowed
- The type system proves that if your code compiles, there are no data races
- Actors are the unit of concurrency—each actor has its own heap and can only communicate via messages
- The garbage collector runs per-actor, not globally, making pause-free concurrency possible

This means you get actor-based concurrency with the performance of manual memory management and the safety of a garbage collector—without the usual tradeoffs.

## Code Example

```pony filename="pony_actors.pony"
actor Main
  new create(env: Env) =>
    // Create two actors that work concurrently
    let reader = Reader
    let writer = Writer
    
    // Send messages (asynchronously)
    reader.read()
    writer.write("Hello from Pony!")
    
    // Actors process messages in their own queue
    // No locks needed - data races are impossible

actor Reader
  var _data: String = ""
  
  be read() =>
    // 'be' marks this as a behavior (async method)
    _data = "Data loaded"
    env.out.print("Reader got: " + _data)

actor Writer
  var _buffer: String = ""
  
  be write(msg: String) =>
    _buffer = msg
    env.out.print("Writer stored: " + _buffer)
```

## Did You Know?

<Callout type="info" emoji="🤔">
  Pony's garbage collector is stop-the-world free at the actor level—each actor pauses its own garbage collection independently, never causing global GC pauses. This makes Pony programs have consistent, predictable latency even under heavy concurrent load.
</Callout>
