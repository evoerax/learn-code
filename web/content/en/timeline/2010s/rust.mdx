## Origin Story

Graydon Hoare created Rust in 2006 while working at Mozilla Research, naming it after the robustust fungus—a tribute to its ability to survive harsh conditions. The language was officially sponsored by Mozilla beginning in 2009 and the first stable release came in 2010.

Rust was born from a practical problem: how to write low-level systems code without the memory bugs that plague C and C++. Classic issues like dangling pointers, buffer overflows, and data races were costing the industry billions in security vulnerabilities and crashes.

The breakthrough was the ownership system—a set of rules the compiler enforces at compile time, eliminating entire categories of bugs without garbage collection. Programs can run with deterministic performance (no GC pauses) while being memory-safe.

## Design Philosophy

<Callout type="default" emoji="💡">
  "The goal of Rust is to give programmers the low-level control they need, while preventing the bugs that low-level control historically introduces." — Graydon Hoare
</Callout>

Rust's core innovation is the ownership and borrowing system:

- **Ownership**: Every value has exactly one owner; when the owner goes out of scope, the value is freed
- **Borrowing**: You can reference values without taking ownership, with the compiler enforcing that either multiple immutable references OR one mutable reference exist (never both)
- **Lifetime annotations**: The compiler tracks how long references are valid, preventing dangling pointers

This means Rust can catch use-after-free, double-free, and data races at compile time—before the program ever runs.

## Code Example

```rust filename="rust_ownership.rs"
// Rust's ownership system prevents memory bugs at compile time

fn main() {
    // Ownership transfer (move semantics)
    let s1 = String::from("hello");
    let s2 = s1; // s1 is moved to s2, s1 no longer valid
    // println!("{}", s1); // ERROR: value borrowed after move
    
    // Borrowing (reference)
    let s3 = String::from("world");
    let len = calculate_length(&s3); // pass reference, not ownership
    println!("{} has {} characters", s3, len); // s3 still valid
    
    // Mutable borrowing
    let mut counter = 0;
    let inc = &mut counter;
    *inc += 1;
    // Can't have another mutable reference while inc is live
    
    // Pattern matching with ownership
    let data = Some(vec![1, 2, 3]);
    match data {
        Some(vec) if vec.len() > 2 => println!("Got {} items", vec.len()),
        _ => println!("Not enough data"),
    }
}

fn calculate_length(s: &String) -> usize {
    s.len() // borrow s, don't take ownership
}
```
