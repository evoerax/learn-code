# 2010s — Safety, Types, and Modern Tooling

The 2010s responded to the failures of the previous decades. Buffer overflows, null pointer exceptions, race conditions, and billion-dollar bugs drove a generation of language designers to ask: what if the type system could *prevent* these errors at compile time? Rust eliminated memory safety bugs without garbage collection. TypeScript made large JavaScript codebases manageable. Swift and Kotlin replaced Objective-C and Java with safer, more expressive alternatives.

## Key Themes

**Memory safety becomes a priority.** Rust's ownership and borrowing system was a theoretical breakthrough made practical. The key insight: ownership can be tracked statically, without runtime overhead. A value has one owner; references must be valid; data races are impossible. The Rust compiler rejects programs that violate these rules — before they run.

**Null safety goes mainstream.** The "billion-dollar mistake" — Tony Hoare's term for inventing null references in 1965 — finally started being undone. Kotlin's nullable types, Swift's Optional, and Rust's `Option<T>` all require programmers to explicitly handle the absence-of-value case. Null pointer exceptions became preventable.

**TypeScript saves JavaScript.** As JavaScript codebases grew to millions of lines, the lack of types became a serious engineering problem. TypeScript's gradual type system could be adopted incrementally — you could add types to existing JavaScript a file at a time. By 2020, TypeScript was arguably more widely used than JavaScript itself.

**Functional languages reach mobile.** Elm proved that purely functional programming could make frontend development more reliable — the Elm Architecture influenced React's design. ReScript and PureScript brought type safety to JavaScript interop. Elixir brought Erlang's battle-tested concurrency model to developers who found Erlang's syntax daunting.

**Blockchain creates new demands.** Solidity introduced a new class of correctness concerns: smart contracts can't be patched after deployment, and bugs can cost millions. This drove intense interest in formal verification and type systems that could prove contract properties.

## Looking Forward

The 2010s ended with memory safety and type safety firmly in the Overton window of mainstream programming. The 2020s would continue this trend — new languages from Google, Apple, and the open-source community would make safety a default, not an optional extra.
