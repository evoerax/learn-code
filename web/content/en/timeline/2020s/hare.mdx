## Origin Story

Hare was created by Drew DeVault, known for his work on the suckless.org projects and the Wayland compositor sway. Frustrated with the complexity of modern programming languages, DeVault designed Hare as a return to fundamentals—a language that does one thing well without layers of abstractions.

First released in 2022, Hare prioritizes readability and hackability over feature count. The language has no generics, no inheritance, no exceptions, and no hidden control flow. Every line of Hare code should be understandable by a programmer who has never seen the codebase before.

## Design Philosophy

Hare's core philosophy is **radical simplicity**. The language has a small, comprehensible standard library, a deterministic build system, and a culture that values "doing one thing well" over "supporting everything."

The language ships with a complete, auditable standard library—no third-party dependencies required for most tasks. Memory is managed manually with explicit allocators, but the language provides safe patterns to make this tractable. The result is a language where you can understand the entire stack from hardware to application.

## Code Example

Hare's straightforward syntax and manual memory management:

```hare
use fmt;
use os;
use strings;

fn main() void = {
	let s: str = "Hello, world!";
	fmt::println(s)!;

	// String slicing is explicit and safe
	let slice: str = strings::slice(s, 0, 5);
	fmt::println(slice)!; // "Hello"

	// Manual memory but safe patterns
	let alloc = mem::alloc(int::size);
	defer mem::free(alloc);
	*alloc = 42;
	fmt::println(*alloc)!;
};
```

## Influence & Legacy

**Inspired by:** C, Plan 9, Old Unix philosophy

**Influenced:** (Still early, influencing conversations about language complexity)

Hare represents a reaction against the ever-growing complexity of modern languages. Whether it achieves widespread adoption or not, its existence challenges the assumption that languages need generics, object-oriented inheritance, and massive standard libraries to be useful.
