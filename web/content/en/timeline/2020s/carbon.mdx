## Origin Story

Carbon was announced by Google in July 2022 as an experimental successor to C++. Led by Chandler Carruth, the project aimed to address C++'s technical debt accumulated over four decades while maintaining bidirectional interoperability with existing C++ code. The team acknowledged that evolving C++ itself is nearly impossible due to its massive existing codebase and backwards compatibility requirements.

The goal is not to replace C++ overnight, but to provide a path forward for new C++ projects that eventually want to migrate. Carbon explores modern language design—generic types, memory-safe defaults, enforced visibility—while being able to call into and be called from C++ code directly.

## Design Philosophy

Carbon prioritizes **performance and interoperability** while improving on C++'s known footguns. The language introduces "checked" semantics (like Rust's Result) for error handling, enforces privacy by default, and makes generics first-class citizens rather than an afterthought.

The language is explicitly designed to be incrementally adoptable—you can start using Carbon in a single file within a C++ project and grow from there. This "path to migration" philosophy acknowledges that the world's C++ code isn't going anywhere.

<Callout type="warning" emoji="⚠️">
  Carbon is still in early experimental stages. The language design is not finalized and breaking changes are expected. Don't use it for production systems yet.
</Callout>

## Code Example

Carbon's generics and checked error handling:

```carbon
package MyPackage api;

fn Factorial[n:!Integer](n: i32) -> i32 {
  var result: i32 = 1;
  for (var i: i32 = 2; i <= n; i += 1) {
    result *= i;
  }
  return result;
}

fn ReadFile(path: String) -> String!Error {
  var file: File = open(path);
  defer close(file);
  return read(file);
}

fn Main() {
  var result: i32 = Factorial(5);
  print("5! = {0}", result);
}
```

## Influence & Legacy

**Inspired by:** C++, Rust, Swift, Kotlin

**Influenced:** (Early days, but intended to influence future systems programming language design)

Carbon represents Google's vision for the future of systems programming. Whether or not Carbon itself succeeds, its experiments with C++ interoperability and migration paths will inform how future languages approach the problem of replacing legacy codebases.
