# 1970s — Systems and Semantics

The 1970s split into two directions that still define programming today. On one side: C and Unix, embracing low-level control and simplicity. On the other: ML, Scheme, and Smalltalk, exploring type theory, functional programming, and pure object orientation. The decade gave us the tools to build operating systems — and the theory to understand what programs mean.

## Context

| | |
|---|---|
| **Era** | Unix, structured programming, type theory |
| **Key hardware** | PDP-11, VAX 11/780, early microprocessors |
| **Milestone** | Unix and C co-developed at Bell Labs (1969–1973) |
| **Key challenge** | Building reliable, portable systems software |

<Callout type="info">
  C and Unix were developed together — each shaping the other. Unix needed a portable systems language; C needed a real system to be written in. The result was an operating system that could be ported to new hardware by rewriting a small C core, not rewiring circuits.
</Callout>

## Key Themes

**C's radical simplicity.** Dennis Ritchie designed C to be small enough that a programmer could hold the entire language in their head. No objects, no garbage collection, no exceptions — just control flow, functions, and pointers. This simplicity made C portable and fast, but also dangerous: buffer overflows, dangling pointers, and undefined behavior are C's enduring legacy.

**Type theory enters practice.** ML was developed as a meta-language for theorem proving, but its type system was a revelation. Hindley-Milner type inference let the compiler deduce types without annotations — you wrote code, the type checker found errors. Parametric polymorphism (generics) was formalized here. ML's type system influenced every statically typed language built after 1980.

**Objects done right.** Alan Kay's Smalltalk at Xerox PARC was the purest expression of the object-oriented idea: everything is an object, all computation is message-passing, and the runtime environment is a live, interactive system. Smalltalk invented the modern IDE with syntax highlighting, method browsers, and integrated debuggers.

**Data query becomes declarative.** SQL asked a revolutionary question: what if you described *what* data you want, not *how* to retrieve it? The relational model and SQL query language decoupled application logic from storage — a separation of concerns that proved so powerful it still dominates data management today.

## Looking Forward

The 1970s gave programmers their most powerful tools yet. The 1980s would take object-oriented programming mainstream, add it to C, and watch the software industry transform.
