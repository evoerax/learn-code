# 1990s — The Internet Age

The 1990s transformed programming from a specialized craft into a global industry. The World Wide Web created an urgent demand for languages that could handle networking, dynamic content, and rapid development. Python, Java, JavaScript, Ruby, and PHP emerged in this decade — and fundamentally changed who could program, what software could do, and how quickly it could be built.

## Key Themes

**The web changes everything.** Before the 1990s, programming was about single machines or local networks. The web introduced stateless protocols, HTML rendering, client-server architecture, and the need for rapid iteration. PHP was literally "Personal Home Page Tools." JavaScript ran inside browsers. Java applets promised rich web content. The internet made programming accessible to anyone with a browser and a text editor.

**Java's bold promise.** "Write Once, Run Anywhere" was the killer proposition. Java compiled to bytecode that ran on the JVM, which was ported to every platform. This solved the deployment nightmare of the 1980s — when you had to recompile C++ for every target OS. Java became the default choice for enterprise backends, Android apps, and large-scale systems. The JVM itself became a platform that later languages (Scala, Clojure, Kotlin) would target.

**Python's quiet rise.** Python was designed with a radical idea: code should be readable. Mandatory indentation, English keywords, and a "batteries included" standard library made it the most beginner-friendly language that was also powerful enough for serious work. It took two decades for the world to catch up, but Python's trajectory from 1991 to today is the story of a language that bet on developer experience and won.

**Ruby and developer happiness.** Yukihiro Matsumoto explicitly designed Ruby to make programmers happy. "We often focus on the machine's needs," he said, "but I wanted to focus on the human's needs." This philosophy would culminate in Ruby on Rails (2004), which proved that a language designed for joy could also be enormously productive for business.

**Functional programming enters the mainstream.** Haskell was the first language to combine pure functional programming with lazy evaluation and a powerful type-class system. Though not widely adopted in industry initially, Haskell's research output — monads, type classes, lazy evaluation — influenced every major language that followed. OCaml brought ML's type inference to a practical, object-capable language.

## Looking Forward

The 1990s ended with the dot-com boom at full speed. The web had proven that interpreted, dynamically typed languages could power global businesses. The 2000s would respond with a more pragmatic phase: languages that combined the best ideas from multiple paradigms, targeted specific platforms (JVM, .NET), and addressed the scaling challenges that web success had created.
