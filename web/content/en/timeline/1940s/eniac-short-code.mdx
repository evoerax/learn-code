## Origin Story

ENIAC Short Code was developed in 1946 by John Mauchly and J. Presper Eckert at the University of Pennsylvania. It was the first programming language to actually run on an electronic computer—the ENIAC (Electronic Numerical Integrator and Computer).

Before ENIAC Short Code, computers were programmed directly in machine code, using physical switches and cables. Short Code introduced the revolutionary concept of representing mathematical expressions in a symbolic form that a computer could interpret and execute.

The language was technically an interpreted system—programs were written using mathematical notation and then manually converted (by a team of women known as the "ENIAC programmers") into machine code that the ENIAC could execute.

## Design Philosophy

The fundamental innovation of ENIAC Short Code was abstraction: replacing the tedious process of direct machine programming with a symbolic notation closer to mathematical conventions.

<Callout type="default" emoji="💡">
  ENIAC Short Code represented numbers using binary-coded decimal and required manual compilation by programmers who would encode instructions onto punch cards.
</Callout>

Programs were written on sheets where each statement contained a variable name, an operator, and a value. The compiler (humans) would translate these into the ENIAC's machine language.

## Code Example

```shortcode
A = 5
B = 3
C = A + B
```

This simple example shows how ENIAC Short Code allowed programmers to express arithmetic operations in a notation that, while still requiring manual compilation, was far more accessible than raw machine code.

## Influence & Legacy

**Inspired by:** Mathematical notation, human computation methods

**Influenced:** All subsequent programming languages

ENIAC Short Code established the radical idea that computers could be programmed using symbolic notation rather than physical reconfiguration. Every programming language that followed—whether assembly, FORTRAN, or Python—owes its existence to this pioneering work in abstraction.

The concept of separating the programmer's intent (in symbolic form) from the machine's execution (in binary) became the foundational principle of all software development.
