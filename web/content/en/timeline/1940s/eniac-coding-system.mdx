## Origin Story

The ENIAC Coding System refers to the original method of programming the ENIAC (Electronic Numerical Integrator and Computer) between 1943 and its public unveiling in 1946. Unlike later programming languages, ENIAC was programmed by physically configuring its circuit pathways using patch cords—essentially connecting wires by hand.

The primary motivation was calculating ballistic trajectories for the US Army's artillery tables. The ENIAC team, including John Mauchly and Presper Eckert, developed this programming method to solve the complex differential equations required for accurate artillery fire tables.

<Callout type="default" emoji="💡">
  Programming ENIAC required physically plugging and unplugging hundreds of cables—a process that could take days for complex calculations.
</Callout>

## Design Philosophy

The ENIAC Coding System wasn't a language in the modern sense but rather a hardware configuration methodology. The computer's function was determined entirely by how its components were wired together:

- **Patch Panel Programming**: Operators used a large patch panel to create temporary circuit connections
- **Function Tables**: Sixty constant voltage sources could be set to represent mathematical constants
- **Cycle Storage**: Mechanical counters stored numbers in a decimal accumulation system

This approach was unique because the program's logic was literally hardwired into the machine's physical configuration.

## Influence & Legacy

**Inspired by:** Mechanical calculators, analog computing methods

**Influenced:** ENIAC Short Code, modern programming

The ENIAC Coding System demonstrated that electronic computers could solve complex mathematical problems through programmable configurations. While primitive by modern standards, it established the radical concept that the same hardware could perform different tasks based on how it was "programmed"—the foundation of stored-program computers that would follow.
