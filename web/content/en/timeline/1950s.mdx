# 1950s — The First Languages

The 1950s was the decade that invented programming as we know it. Fortran proved that compilers could generate efficient code. LISP introduced functional programming and garbage collection. COBOL made computers accessible to business. By decade's end, programmers had real languages — not just machine codes.

## Context

| | |
|---|---|
| **Era** | First high-level languages, first compilers |
| **Key hardware** | IBM 701, UNIVAC I, IBM 704, IBM 709 |
| **Milestone** | First compiler (A-0, 1952, Grace Hopper) |
| **Key challenge** | Convincing engineers that compiled code could be as fast as hand-written assembly |

<Callout type="info">
  When Grace Hopper proposed the first compiler in 1952, she was told it was impossible — computers could only do arithmetic, not translate symbols. She built it anyway.
</Callout>

## Languages of the 1950s

| Language | Year | Creator | Significance |
|----------|------|---------|--------------|
| Assembly | 1950s | Various | Human-readable machine instructions; foundation for all languages |
| Autocode | 1952 | Alick Glennie | First compiled language actually implemented and used |
| Fortran | 1957 | John Backus (IBM) | First widely adopted high-level language; compiler efficiency proof |
| FLOW-MATIC | 1955 | Grace Hopper | First English-like business language; direct ancestor of COBOL |
| IPL (Information Processing Language) | 1956 | Newell, Shaw, Simon | First list-processing language; AI research foundation |
| LISP | 1958 | John McCarthy (MIT) | Functional programming, garbage collection, code-as-data |
| COBOL | 1959 | Grace Hopper & committee | Business language designed for readability by non-programmers |

## Key Themes

**The compiler war.** John Backus and his IBM team built Fortran's compiler over three years, with the explicit goal of generating code as good as assembly. They largely succeeded — Fortran's adoption by scientists and engineers was proof that high-level languages were viable.

**Two philosophies emerge.** Fortran and COBOL optimized for existing ways of thinking (math formulas, business English). LISP took a radically different approach: build a minimal, mathematically pure system where programs and data have the same form. These two philosophies — pragmatic vs. principled — would define language design debates for decades.

**AI is born.** IPL and LISP were created specifically to support artificial intelligence research. McCarthy's LISP enabled early AI programs like the General Problem Solver. The tight connection between programming language theory and AI research would persist through the 1980s.

**Standardization begins.** COBOL was designed by a government committee (CODASYL) with military requirements — a language that could run on any vendor's hardware. This standardization effort introduced the idea that languages could be portable across machines.

## Notable Firsts

- First compiler: A-0 System (1952, Grace Hopper)
- First widely-used high-level language: Fortran (1957)
- First garbage collector: LISP (1958)
- First language designed for business: COBOL (1959)
- First list-processing language: IPL (1956)

## Looking Forward

The 1950s established that high-level languages were practical and portable. The 1960s would build on this foundation: structured programming would challenge "goto"-heavy code, object-oriented programming would be conceived, and interactive computing would begin.
