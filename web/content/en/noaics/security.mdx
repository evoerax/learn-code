# Cybersecurity & information security

How to **prevent attacks, enforce authorization, and protect confidentiality and integrity** under compliance and audit constraints.

---

## Sub-areas

| Topic | Typical questions |
| :--- | :--- |
| **Cryptography** | Protocol design, post-quantum, homomorphic & MPC (application-dependent) |
| **Systems security** | OS hardening, sandboxing, TEEs, HSMs |
| **Software & supply chain** | Dependencies, SBOM, reproducible builds, exploitation & mitigations |
| **Web / cloud security** | Identity, authz, multi-tenant isolation, misconfiguration |
| **Privacy** | Differential privacy, anonymization, regulation vs engineering |

---

## 2026 themes

Public discourse often highlights **machine/non-human identities**, **AI agent permissions**, and **IaC / generated-code supply-chain risk**. See e.g. [Cloud Security Alliance](https://cloudsecurityalliance.org/). Security intersects heavily with AI, but **security as a discipline** remains a top-level area in curricula and venue lists.
