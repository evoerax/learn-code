# Software engineering

How to build and maintain **large software systems**: not just coding, but **long-term evolution, observability, rollback, and collaboration**.

---

## Sub-areas

| Topic | Typical questions |
| :--- | :--- |
| **Architecture** | Modularity, boundaries, dependency governance, evolution |
| **Quality & verification** | Testing, static/dynamic analysis, formal methods (ties to theory) |
| **Delivery & operations** | CI/CD, release strategies, canaries, observability |
| **Platform engineering** | Internal developer platforms, golden paths, self-service & governance |
| **Process & collaboration** | Requirements, code review, tech-debt metrics |

---

## 2026 narrative

Industry often frames this lane as **platform engineering**, **developer experience**, and **sustainable delivery**—see Thoughtworks [Looking Glass](https://www.thoughtworks.com/insights/looking-glass).

## Overlap with AI

LLMs change **how** code is written and reviewed; **pipelines, policy, governance, and supply-chain security** remain software-engineering problems.
