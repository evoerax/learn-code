# Databases & data management

How data is stored, queried, and governed at scale with **correctness, cost, and operability**.

---

## Sub-areas

| Topic | Typical questions |
| :--- | :--- |
| **Relational & SQL** | Query optimization, indexing, transactions & isolation |
| **NoSQL / NewSQL** | Scaling models, CAP trade-offs, specialized engines |
| **Distributed databases** | Replication, sharding, consensus, failover |
| **Analytics & lakehouse** | Column stores, materialized views, metadata & governance |
| **Streaming** | Exactly-once, windows, Kafka/Flink-class semantics |
| **Search & specialized indexes** | Full-text, vectors (often co-located with AI apps, but engines remain data-systems work) |

---

## Overlap with AI

**Vector search and feature stores** are hot; **storage engines, logging structures, query optimization, and disaster recovery** remain classic database research. [Datasets](/en/datasets/overview) focuses on **corpora and benchmarks**; this page focuses on **data systems** themselves.
