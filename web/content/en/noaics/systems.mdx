# Computer systems

The performance and reliability core: **how computers run faster, more reliably, with fewer resources**. This page expands the **systems** pillar from the [NOAICS overview](/en/noaics/overview).

---

## Sub-areas

| Topic | Typical questions |
| :--- | :--- |
| **Operating systems** | Scheduling, virtual memory, FS APIs, virtualization, containers vs kernel |
| **Architecture** | ISA, CPU/GPU/accelerators, cache hierarchy, power & reliability |
| **Compilers & runtimes** | Optimization, JIT/AOT, IR pipelines, language implementation |
| **Parallel & distributed** | Consistency, fault tolerance, messaging vs shared memory, schedulers |
| **Storage** | File systems, block devices, NVMe, near-data processing |
| **HPC** | Massive parallelism, interconnects, numerical libraries |

---

## Overlap with AI

Much “AI infra” (training schedulers, inference serving, memory pressure) is **still systems work**. This site keeps **model narratives** under [AI Models](/en/ais/overview) and **frameworks/serving** under [AI Software](/en/software/overview); this page keeps **classical CS systems** framing.
